## 1. Executive summary

Phase 5 is where Sage starts to **reason**: it turns the evidence accumulated in
Phases 1–4 (Company Truth, canonical intel items, lead outcomes, offers,
constraints, executive memory) into a small number of ranked, evidence-linked,
approval-gated **opportunities**; hands approved work to execution departments
through a structured **Directive Bus** (never bypassing their existing approval
flows); records every owner decision in a **decisions table** that closes the
loop from recommendation → decision → execution → measured result → lesson;
explains week-over-week performance changes with a deterministic **Change
Diagnostics** engine (arithmetic first, AI narrates only the computed numbers);
and makes the accumulated knowledge visible on a **"What Sage knows"** page.

Everything is advisory by default: Sage proposes, the owner decides, departments
execute through their existing gates. The Phase 4 inert constraint clamp gets
its two designed enforcement points — both of which only ever *shrink or flag*
what Sage proposes; they never spend, publish, or act. Four new feature flags,
all default OFF; flag-dark remains byte-identical. **Recommendation at the end:
Ready to implement, pending your approval.**

## 2. Exact approved Phase 5 scope

1. **Opportunity queue** — `sage_opportunities` + junction tables, generated by
   a weekly Opportunity Synthesis job; **bucket-ranked** (no numeric scores);
   max **5 open proposals** per brand; the Monday briefing presents at most
   **3 decisions**; everything else waits or expires.
2. **Directive Bus** — `sage_directives`: structured handoffs of *approved*
   opportunities to Nova/Atlas/Forge/Pulse/Voice through their existing entry
   points; departments write results back; a nightly join fills
   `measured_result` on the parent opportunity.
3. **Decisions table** — `sage_decisions`: one row per owner decision on
   anything Sage proposed, with rationale, execution status, measured result,
   outcome, and lesson; lessons distilled into learnings that feed future
   synthesis prompts.
4. **Change Diagnostics** — deterministic week-over-week decomposition
   (Δleads = Δspend effect + ΔCTR effect + Δconversion effect + mix shift) from
   existing `analytics` + `leads` history; AI narrates the arithmetic, never
   invents causes; feeds the briefing's "what changed" section.
   **AI-call accounting rule (W7, locked):** the decomposition itself is pure
   SQL/arithmetic; its narration is produced **inside the single weekly
   Opportunity Synthesis call** (the computed terms are part of that prompt's
   input, and the narrative is one field of its validated output). Diagnostics
   is NEVER a separate recurring AI job — if synthesis is skip-gated or fails,
   `narrative` stays NULL and the deterministic numbers still render.
5. **"What Sage knows" page** — owner-facing render of approved Company Truth,
   executive memory, offers, constraints, and active intel items with
   confidence labels and an export; the trust surface.
6. **Constraint clamp wiring** — the Phase 4 inert helper connects at exactly
   two points: Opportunity Synthesis post-validation and Atlas directive budget
   clamps (§8).

Explicitly **NOT** in Phase 5 (Phase 6+ — do not build): channel scorecards,
forecasts, Executive Debate, Top-3-bets strategy object, self-eval scorecard,
experiments, playbooks, trust-graduation auto-approval, cross-brand aggregates,
"ask Sage anything" Q&A.

## 4. Database additions (one migration, `120_sage_v2_phase5.sql`, additive only)

All idempotent (`IF NOT EXISTS`), CHECK-constrained lifecycles, real FKs, no
uuid[] arrays (W3). No existing table or column is altered.

- **`sage_opportunities`** — `opportunity_id` PK, `brand_id` FK, `title`,
  `thesis`, `confidence` (`verified|reported|inferred` = min of its evidence),
  `expected_impact_cents` BIGINT NULL + `impact_basis` TEXT (estimate must state
  its basis or stay null — never fabricated), `cost_estimate_cents` NULL,
  `effort` (`s|m|l`), `risk` TEXT, `category` (owner-relevant grouping),
  `recommended_department` (`nova|atlas|forge|pulse|voice|owner`),
  `success_metric` / `failure_metric` (metric+threshold+window jsonb where
  machine-checkable), `constraint_flags` JSONB (clamp results, §8),
  `rationale` JSONB (§10 — the explainability record), `expires_at`,
  `status` CHECK: `proposed → approved | declined | expired → directed →
  in_progress → executed → measuring → succeeded | failed | inconclusive`,
  `owner_decision_note`, `measured_result` JSONB, `lesson` TEXT, timestamps.
  Partial index enforcing max-5-open is checked in code (count under the
  per-brand advisory lock at insert; a DB CHECK can't count rows).
- **`sage_opportunity_evidence`** — `(opportunity_id FK, item_id FK →
  sage_intel_items)` junction, PK on the pair. ≥1 row required per opportunity
  (enforced in the single creation chokepoint: **no evidence, no opportunity**).
- **`sage_opportunity_deps`** — `(opportunity_id, depends_on_id)` junction.
- **`sage_directives`** — `directive_id` PK, `opportunity_id` FK, `brand_id`,
  `department`, `instruction` JSONB (department-specific schema, validated
  before insert), `status` CHECK `issued → acknowledged → done | failed`,
  `result` JSONB (department writes back), `clamp_applied` JSONB (§8),
  timestamps. Unique partial index: one active directive per
  (opportunity, department).
- **`sage_decisions`** — `decision_id` PK, `brand_id`, `subject_type`
  (`opportunity` — the only P5 subject; enum widens in later phases),
  `subject_id`, `decided` (`approved|declined|revised`), `decision_via`
  (`briefing|opportunities_tab|voice`), `why` TEXT (owner's words, optional),
  `executed` BOOL, `measured_result` JSONB, `outcome`
  (`worked|failed|unknown`), `lesson` TEXT, timestamps.
- **`sage_change_diagnostics`** — `diag_id` PK, `brand_id`, `week_start` DATE,
  `terms` JSONB (each decomposition term: name, value, inputs used),
  `data_coverage` JSONB (which inputs existed), `narrative` TEXT NULL (AI
  narration; NULL when AI unavailable — the numbers still render),
  `input_hash`, UNIQUE `(brand_id, week_start)`.

Rollback note: with flags OFF nothing writes these tables; they can sit empty
indefinitely (the proven P2–P4 pattern).

## 6. Decision flow and data flow

```
EVIDENCE (read-only inputs)
  Company Truth (approved only) · sage_intel_items · leads outcomes/attribution
  · analytics history · sage_offers (internal audience incl. margin notes)
  · brand_constraints · sage_memory · sage_data_quality_flags · past
  sage_decisions + lessons
        │  Monday, per brand, via sage_job_queue + input-hash skip gate
        ▼
CHANGE DIAGNOSTICS (deterministic arithmetic → terms JSONB; AI narrates only)
        ▼
OPPORTUNITY SYNTHESIS (one Claude call, internal audience)
  → candidate opportunities, each with thesis + evidence item ids + rationale
  → code-side validation: evidence ids must exist & belong to the brand
    (no evidence, no opportunity); impact estimate must carry a basis or be
    nulled; constraint clamp post-validation (§8); max 5 open enforced
        ▼
OWNER DECISION (advisory gate — nothing proceeds without it)
  Monday briefing surfaces ≤3 · Opportunities tab shows the rest
  approve / decline / revise  →  sage_decisions row (always)
        ▼
DIRECTIVE BUS (approved + department-recommended only)
  sage_directives row with validated instruction JSONB (Atlas budgets clamped)
  → department consumes via its EXISTING entry point & approval flow
  → department writes result back
        ▼
MEASUREMENT (nightly deterministic join) → measured_result on opportunity
        ▼
DECISION REVIEW  → outcome + lesson on sage_decisions → distilled learnings
  → injected into next synthesis prompt (the loop closes)
```

Owner-directed opportunities (`recommended_department='owner'`) stop at the
decision: no directive is ever issued for them.

## 7. Opportunity Synthesis — how it works

- **Cadence:** weekly (Monday, before the briefing), per brand, enqueued on
  `sage_job_queue`, claimed `FOR UPDATE SKIP LOCKED`, input-hash skip-gated
  (unchanged evidence ⇒ zero AI calls, logged `skipped_unchanged`; the
  briefing still renders the standing queue — skip gates suppress cost, never
  owner-facing deliverables).
- **Prompt:** internal audience via the Phase 4 context chokepoint; Company
  Truth first and highest priority; then diagnostics terms, outcomes (with
  coverage %), intel items (sensitive items excluded from any cross-brand use;
  owner-only), offers (incl. margin notes — internal), constraints, memory,
  active data-quality flags, and distilled lessons from past decisions.
- **Output contract (validated, AI failure ⇒ job marked failed, never
  fabricated):** each candidate = thesis, category, evidence item ids (≥1),
  confidence claim (code recomputes as min of cited evidence — the model
  cannot award itself confidence), impact estimate + basis or null, effort,
  risk, department, success/failure metric, rationale fields (§10).
- **Ranking (W4 — transparent bucket sort, zero numeric scores):**
  1. evidence confidence tier (verified > reported > inferred),
  2. expiry urgency (sooner first),
  3. owner-relevant category order.
  The UI states the reason in words. Ties beyond that: newest evidence first.
- **Caps:** max 5 `proposed` per brand (counted under the per-brand advisory
  lock at insert; overflow candidates are dropped lowest-bucket-first and the
  drop is logged in the run record). Briefing shows ≤3. Unactioned proposals
  expire at `expires_at` (default 21 days) via the nightly deterministic sweep
  — expiry is a status flip + decision-history entry, never a re-ask.
- **Dedup:** a stable content key (brand + category + normalized thesis stem)
  with a partial unique index on open statuses — the same opportunity is never
  proposed twice while one is open, and a declined one cannot reappear within
  90 days unless new evidence (later `created_at` than the decline) cites it,
  in which case the rationale must say what's new.

## 8. Connecting the inert constraint clamp (Phase 4 → Phase 5)

`utils/constraintClamp.js` is wired at **exactly the two points named in its
header comment** — nowhere else:

1. **Opportunity Synthesis post-validation.** After parsing AI output, before
   insert: `clampBudget(cost_estimate, monthly_budget, committed_this_month)`
   (committed = approved-but-unfinished opportunity costs + active directive
   budgets this month, computed deterministically), `checkCapacity(projected
   weekly volume, weekly_capacity)`, `isBlackedOut(proposed window)`. Results
   land in `constraint_flags` and the rationale ("fits your $2,000/mo budget:
   $1,400 remaining"). A violating opportunity is **not silently dropped** —
   it is either downscaled (with the clamp noted) or presented flagged
   ("exceeds your stated capacity — approve only if you can staff it").
   Advisory-honest: the owner always sees the conflict; Sage never hides it
   and never overrides the owner's stated constraint on its own.
2. **Atlas directive budget clamp.** When issuing an Atlas directive,
   `instruction.budget_cents = clampBudget(...).allowedCents`; the original
   ask and clamp reason are stored in `clamp_applied`. A fully-clamped-to-zero
   budget blocks the directive with a visible reason instead of issuing a $0
   instruction.

The clamp remains pure and unchanged; unknown constraints still mean **no
clamp** (unknown ≠ zero). It shrinks or flags proposals; it never initiates,
spends, or acts. Constraints gain no other enforcement surface — Autopilot,
manual campaigns, and every existing flow are untouched.

## 9. Atlas directives — generation and constraint

- Generated **only** from an owner-approved opportunity whose recommended
  department is Atlas; instruction schema `{budget_cents, audience, geo,
  creative_brief_ref, window}` validated before insert.
- Budget clamped (§8.2); geo passes through the existing geo-targeting
  exclusion rules (hard blocks preserved — memory rule); window checked
  against blackout dates.
- Atlas consumes the directive through its existing campaign-creation path,
  which retains ALL current guardrails (Autopilot spend limits, approval
  queue where configured). **A directive is work-in-an-inbox, not an
  execution command** — if Atlas's own flow requires owner approval, the owner
  approves twice (strategy, then execution); Phase 5 does not relax any
  existing gate.
- Same pattern for Nova/Forge/Pulse/Voice with their own instruction schemas;
  Atlas is called out only because it is the one that touches money.

## 10. Explainability and audit trail

Every opportunity carries a structured `rationale` JSONB written at creation
and immutable thereafter:

- `because[]` — evidence citations: item id, source, date, confidence, and the
  one-line claim each contributes ("Ad Spy, verified, Jul 12: competitor
  launched 20%-off financing offer").
- `derived_from` — which context blocks informed it (truth section ids,
  diagnostics terms, outcome stats WITH their coverage %, memory ids, offer
  ids, constraint fields) — ids only, so the display layer re-renders from
  live rows and deleted evidence shows as "evidence no longer available",
  never silently re-written.
- `constraint_flags` — every clamp result (§8).
- `honesty` — data gaps the synthesis knew about ("outcome coverage 22% —
  impact estimate withheld").

The UI's "Why is Sage recommending this?" expands this record verbatim. The
audit chain is: intel item (immutable, source-linked) → junction row →
opportunity + rationale → decision row (who/when/why/via) → directive +
result → measured_result → lesson. Every hop is a FK; "show me why" is a
join, not an archaeology dig. AI usage for synthesis/narration is logged to
the existing AI ledger per brand+feature, making cost-per-approved-
recommendation computable (the P5 efficiency metric from the locked plan).

## 11. Privacy, honesty, and safety safeguards

- **Advisory by default (CEO principle):** nothing executes without an owner
  approval; owner-department items never leave the queue; departments keep
  their own gates. There is NO autonomous action in Phase 5.
- **Evidence-based by construction:** no evidence row, no opportunity —
  enforced in the single creation chokepoint, not the prompt.
- **No fabricated numbers:** impact estimates need a stated basis or stay
  null; below 30% outcome coverage financial estimates are withheld (P3 rule
  reused); confidence is computed from evidence, never model-asserted.
- **Sensitive intel items** (P2 flag) stay owner-only and are excluded from
  anything cross-brand — unchanged and re-asserted in the synthesis query.
- **Customer allowlist untouched:** synthesis is internal-audience; the Phase
  4 allowlist for customer-facing prompts is not modified by Phase 5, and the
  existing sentinel leak tests must still pass unchanged. Opportunities,
  rationales, diagnostics, and directives NEVER enter customer-facing prompts.
- **Demo brands excluded** at the data-gathering layer (house rule).
- **AI failure ⇒ 502 / failed job**, visible in admin; diagnostics renders its
  deterministic numbers even when narration fails; the briefing says "Sage
  couldn't complete this week's analysis" rather than inventing content.
- **PII:** rationale cites intel items by id; the P2 ingestion redaction
  already scrubbed them. Owner decision notes are owner-authored and
  owner-visible only.
- **Concurrency:** queue claims, advisory-lock cap counting, status-guarded
  terminal updates, stale-claim rescue marking failed (never auto-retrying AI
  work) — all existing house patterns.

## 13. Testing strategy

- **Unit (node:test, DB-stubbed per house pattern):** bucket-sort ordering +
  tie-breaks; evidence-required rejection; confidence = min(evidence);
  impact-basis-or-null; max-5 cap under simulated concurrency; dedup /
  declined-90-day re-propose rules; clamp wiring (downscale vs flag vs block,
  unknown ⇒ no clamp); instruction schema validation per department;
  diagnostics decomposition arithmetic on fixture analytics (terms sum to the
  observed delta; missing weeks ⇒ honest coverage gaps, never zeros);
  decision atomicity (status-guarded UPDATE row-count branch); expiry sweep;
  flag-dark: every endpoint `{enabled:false}`, zero Phase 5 table queries,
  synthesis/diagnostics jobs no-op.
- **Allowlist regression:** Phase 4 sentinel leak tests re-run unchanged; new
  sentinel test proving opportunity/rationale/directive content never appears
  in a customer-audience context build.
- **Loop test:** seeded evidence → synthesis (AI stubbed) → approve →
  directive → result write-back → nightly join fills measured_result →
  lesson lands on the decision row.
- **Client (vitest):** Opportunities tab render/decide/hide-on-disabled;
  rationale expansion renders citations; knowledge page confidence labels;
  briefing decisions card.
- **Gates:** existing `test`, `client-test`, `client-build` must stay green;
  full suites + architect self-review before the completion report, same as
  Milestones 1–4.

## 14. Rollback strategy

- All four flags OFF restores today's behavior byte-for-byte (tables sit
  empty/ignored; no existing code path is conditioned on Phase 5 data).
- Migration 120 is additive-only; rollback = flags off, tables dormant. No
  destructive change to revert.
- Directives half-executed at flag-off: departments finish what their own
  flows already accepted (their gates were honored), the nightly join is
  dark, statuses freeze honestly — nothing new is issued.
- Each flag can be disabled independently (e.g. keep diagnostics, pause
  opportunities) because no Phase 5 feature assumes another is ON.

## 15. Risks and mitigations

| Risk | Mitigation |
|---|---|
| Synthesis produces plausible-but-thin opportunities | evidence-required chokepoint + confidence recomputation + max-5 scarcity + bucket ranking favoring verified evidence |
| Approval fatigue returns | weekly-3 briefing cap is hard; tab is pull-only; expiry sweep clears stale asks (W5) |
| A fifth Monday narrative sneaks in | diagnostics + decisions render INSIDE the existing briefing; no new report artifact (W6 hard rule) |
| Directive double-execution | unique active directive per (opportunity, department) + atomic claim + status-guarded transitions |
| Clamp wrongly blocks with bad data | clamp is fail-open on invalid inputs (unknown ≠ zero, already tested); flags surface, owner decides |
| Measurement join mis-attributes results | join is deterministic on directive-linked entities only; unlinked performance is never claimed; `unknown` outcome is a first-class honest state |
| Cost creep | one weekly AI call per brand, skip-gated; narration optional; cost per approved recommendation tracked from the ledger |
| Scale (W7) | queue-claimed, no new recurring per-brand AI jobs beyond the weekly synthesis |

## 16. Architectural conflicts identified (report-before-change)

1. **Blueprint §7 vs challenge review:** the blueprint's `evidence_item_ids
   uuid[]`, numeric priority score, and nightly recompute are all **replaced**
   by junction tables and bucket ranking per W3/W4 — this document follows the
   locked revision, not the original blueprint.
2. **Strategy Engine (`sage_strategy` table, quarterly review)** appears in
   blueprint §7 but the locked P5 row does not include it; Top-3-bets is P6.
   **Excluded from Phase 5.** Flagging so its absence is a decision, not an
   omission.
3. **Autopilot overlap:** Autopilot already proposes weekly content/ad batches.
   Boundary: Autopilot = execution cadence (what to publish this week); Sage
   opportunities = strategy moves (where to point effort/money). A Sage→Nova/
   Atlas directive *feeds* Autopilot's existing queue; it never creates a
   second approval surface for the same item. Sage never proposes an
   opportunity that is merely "run Autopilot's next batch."
4. **Self-Review (admin) overlap:** Sage's measured results will eventually
   feed the admin self-eval scorecard — that is P6; P5 only stores the data.
5. **`blackout_dates` format:** JSONB as approved in your Phase 4 §6.3
   decision; the clamp already consumes that shape. No change needed.

## 17. No Phase 6 functionality — confirmation
