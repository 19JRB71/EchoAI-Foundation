Here's Prompt 36: AI Content Calendar and Auto-Posting Scheduler for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system including native mobile app preparation. Now you need to add an AI Content Calendar and Auto-Posting Scheduler that automatically plans, generates, and publishes social media content across all platforms on a consistent schedule without the business owner having to think about it.
In the prompts folder create a content calendar prompt file that defines the AI Content Calendar Agent. This agent accepts a brand profile, the business type, posting frequency preferences, and a content theme or focus for the month. It generates a full thirty day content calendar with a unique post for every scheduled day, each tailored to the correct platform, written in the brand's voice, with suggested posting times based on when that platform's audience is most active. It varies content types across educational posts, promotional posts, behind the scenes posts, customer success stories, tips and tricks, and calls to action. No two consecutive posts should feel the same. The calendar should feel like it was planned by a professional social media manager who deeply knows the brand.
In the controllers folder create a content calendar controller with the following functions. First a generate calendar function that accepts brand ID, posting frequency such as daily, three times per week, or five times per week, selected platforms, and an optional content theme, calls the Anthropic API with the content calendar agent prompt, and returns a complete thirty day calendar with all posts planned out. Second a save calendar function that saves the generated calendar and all its posts to the database. Third a get calendar function that returns the current calendar for a brand with all posts and their status. Fourth an activate calendar function that activates the calendar so the auto-posting scheduler starts publishing posts automatically at their scheduled times. Fifth a pause calendar function that pauses auto-posting without deleting the calendar. Sixth a regenerate post function that regenerates a single post in the calendar if the business owner does not like it.
In the models folder create a migration file that adds a content calendars table with columns for calendar ID, brand ID, month, year, posting frequency, content theme, status as draft, active, or paused, and timestamps. Update the existing social posts table to add a calendar ID foreign key so posts can be linked to their calendar.
Update the existing social media scheduler that runs every minute to check for calendar posts that are due and publish them automatically using the existing social media posting infrastructure.
In the client source folder update the existing Social Media section to add a new Content Calendar tab at the front. This tab shows a full monthly calendar grid view. Each day shows the planned posts with platform icons and a preview of the post content. Days with no posts are clearly marked. Show a Generate Calendar button at the top that opens a setup form asking for posting frequency, platforms, and optional content theme. Show Activate, Pause, and Regenerate buttons. When a post is clicked show the full post content in a side panel with an option to edit before it goes live.
Wire the calendar activation into the existing social media scheduler so active calendar posts publish automatically at their scheduled times.
Rebuild the React client after making changes. Output all files created.
