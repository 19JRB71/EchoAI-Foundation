Here's Prompt 29: AI Phone Agent with Twilio for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system including AI reputation management. Now you need to add an AI Phone Agent powered by Twilio that can make outbound calls to hot leads and receive inbound calls on behalf of the business owner.
In the prompts folder create a phone agent prompt file that defines the AI Phone Agent. This agent conducts natural spoken phone conversations. For outbound calls to hot leads it introduces itself as calling on behalf of the business, references what the lead discussed in their chatbot conversation, and attempts to book an appointment or close the sale. For inbound calls it answers as the business, qualifies the caller using the same tire kicker, warm, and hot scoring system as the chatbot, captures their contact details, and routes urgent calls by sending the business owner an immediate SMS alert. The agent's voice and personality matches the brand profile exactly. It speaks naturally, listens carefully, and never sounds robotic.
In the controllers folder create a phone controller with the following functions. First an initiate outbound call function that accepts a lead ID, retrieves the lead's phone number and conversation history, uses Twilio to place a call, and connects the call to the AI phone agent. Second an inbound call handler function that Twilio calls when someone dials the business phone number, connects the caller to the AI agent, and manages the conversation. Third a call status webhook that Twilio calls when a call ends, saves the call transcript and outcome to the database, updates the lead temperature based on the conversation, and sends an SMS alert to the business owner if the lead is hot. Fourth a get call history function that returns all calls for a brand with their duration, outcome, and transcript summary.
In the models folder create a migration file that adds a calls table with columns for call ID, brand ID, lead ID, call direction as inbound or outbound, caller phone number, call duration, transcript, outcome, lead temperature after call, and timestamps.
Add a Twilio phone number configuration to the settings panel so business owners can connect their Twilio account and assign a phone number to their brand.
In the routes folder create phone routes with POST route for initiating outbound calls, POST routes for the Twilio inbound and status webhooks which require no authentication since Twilio calls them directly, and GET route for call history. Wire all authenticated routes with auth and lockout middleware.
Update server dot js to include phone routes at slash api slash phone. Add Twilio as a dependency. Output all files created.
