Here's Prompt 63: Echo Always-On Voice Conversation for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built Echo with a beautiful ElevenLabs voice. Now you need to make Echo truly conversational — always listening, always ready, responding naturally to voice without any button pressing. This is the feature that makes Echo feel like a real person in the room with you.
PART 1 — WAKE WORD DETECTION
Implement always-on wake word detection using the browser's Web Speech API continuous listening mode. The wake phrase is "Hey Echo." Echo listens passively in the background at all times when the user is logged into EchoAI. The wake word detection uses minimal processing and does not send audio to any server — it runs entirely in the browser for privacy and speed.
When the browser hears "Hey Echo" it activates Echo into full listening mode. A subtle visual indicator appears showing Echo is now actively listening — a soft pulsing green ring around the Echo companion panel. Echo plays a very short audio confirmation tone using ElevenLabs sound effects to indicate it heard the wake word and is ready. Then Echo listens for the user's full message or question.
The wake word detection must be robust enough to distinguish "Hey Echo" from casual conversation that includes the word Echo. It should only trigger on the specific phrase "Hey Echo" spoken directly and clearly.
PART 2 — CONTINUOUS CONVERSATION MODE
After the wake word is detected and the user speaks their message Echo processes it and responds using the ElevenLabs voice. After Echo finishes speaking it automatically stays in active listening mode for 30 seconds waiting for a follow-up response or question. A subtle visual countdown shows how long Echo will stay listening.
If the user responds within 30 seconds Echo processes the response and continues the conversation naturally. If the user does not respond within 30 seconds Echo plays a soft closing tone and returns to passive wake word listening mode.
This creates a natural back and forth conversation. The user says "Hey Echo" once to start and then the conversation flows naturally from there with no further button pressing required. Echo asks questions and waits for answers. The user responds and Echo continues. Just like talking to a real person.
If Echo asks a question it stays in active listening mode indefinitely until the user responds. Echo never stops listening mid-question waiting for an answer. If the user says something like "give me a minute" Echo acknowledges and waits patiently.
PART 3 — MORNING WAKE UP EXPERIENCE
When the user logs in every morning Echo plays a short energetic upbeat music intro generated by ElevenLabs music generation. The music should sound like a punchy AC/DC inspired guitar riff — energetic, bold, and about 3 to 4 seconds long. Generate this music using the ElevenLabs music generation API with a prompt like "short punchy electric guitar riff, AC/DC style, energetic, 3 seconds, wake up music."
After the music plays Echo immediately launches into the morning briefing using the ElevenLabs voice. The transition from music to voice should be seamless — the music fades slightly as Echo begins speaking.
After the morning briefing Echo asks "What would you like to tackle first today?" and immediately enters active listening mode waiting for the user's spoken response. The user just answers naturally and Echo begins executing or planning based on what they said.
PART 4 — ALWAYS ON MICROPHONE WITH PRIVACY CONTROLS
The microphone is always open by default when the user is logged into EchoAI. Build a clear visual indicator in the top navigation bar showing whether Echo is listening. A green microphone icon means Echo is actively listening for the wake word. A pulsing green microphone means Echo is in active conversation mode. A grey microphone with a line through it means Echo is muted.
Clicking the microphone icon toggles between listening and muted. When muted Echo cannot hear anything and the wake word does not work. A muted state persists until the user clicks the microphone icon again or says nothing — mute does not automatically turn off.
Show a subtle animated waveform near the Echo companion panel when Echo is in active listening mode so the user always knows when Echo can hear them.
Store the user's microphone preference in localStorage so if they prefer muted mode it stays muted between sessions.
PART 5 — VOICE COMMAND PROCESSING
When Echo hears a voice command after the wake word it processes the full intent of what was said and takes the appropriate action.
If the user says "Hey Echo what are my leads today" Echo pulls the lead data and speaks the answer.
If the user says "Hey Echo launch the Blacor Homes campaign" Echo confirms with the user before executing and waits for verbal confirmation.
If the user says "Hey Echo switch to South Dixie Storage" Echo switches the brand context and confirms verbally.
If the user says "Hey Echo show me the competitor report" Echo navigates to Scout's department and speaks a summary of the latest competitor intelligence.
If the user says "Hey Echo I need to think about that" Echo acknowledges and waits.
If the user says "Hey Echo mute yourself" Echo says "Going quiet. Say Hey Echo whenever you need me." and mutes itself.
If the user says something Echo does not understand Echo asks for clarification naturally. "I want to make sure I understand you correctly. Could you rephrase that?"
PART 6 — ELEVENLABS SOUND EFFECTS AND PERSONALITY
Use ElevenLabs sound effects to give Echo personality through audio.
When Echo hears the wake word play a short subtle activation sound — a soft chime or click indicating Echo is listening.
When a hot lead comes in play an energetic alert sound before Echo speaks the notification.
When a campaign hits a milestone or a goal is achieved play a brief celebration sound.
When something goes wrong play a subtle alert tone before Echo delivers the bad news.
When Echo is thinking or processing play a very subtle ambient processing sound so the user knows Echo is working.
When the user mutes Echo play a soft goodbye sound.
All sound effects should be tasteful and professional — enhancing the experience without being annoying or distracting.
PART 7 — BROWSER PERMISSION AND FALLBACK
When the user first logs in show a friendly permission request explaining that Echo needs microphone access to have conversations with them. The permission request should feel warm and personal not technical. Something like "Echo would like to listen for your voice so you can have natural conversations without pressing any buttons. Click Allow to enable hands-free conversation or Deny to use the microphone button instead."
If the user denies microphone permission or if their browser does not support continuous listening fall back gracefully to the push-to-talk microphone button that already exists. Show a gentle notification explaining that hands-free mode is not available and offering the button as an alternative.
If the browser loses microphone access mid-session Echo detects this and shows a notification asking the user to re-grant permission.
Wire the wake word detection and conversation mode into the existing Echo companion panel. Wire the morning music and briefing into the existing voice briefing system. Wire voice commands into the existing Echo companion message processing so all existing Echo capabilities are accessible by voice. Use the Web Speech API for wake word detection and continuous listening since it runs in the browser with no server cost. Use ElevenLabs for all audio output including voice responses, sound effects, and morning music. Use the existing OpenAI Whisper integration as a fallback for transcription when needed. Output all files created and updated.
