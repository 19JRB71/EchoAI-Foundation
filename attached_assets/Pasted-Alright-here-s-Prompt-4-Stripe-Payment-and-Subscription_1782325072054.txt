Alright, here's Prompt 4: Stripe Payment and Subscription Management for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've already built the project structure, database schema, and authentication system. Now you need to set up automated billing and subscription management using Stripe.
In the config folder, create a Stripe configuration file that initializes the Stripe SDK using the Stripe secret key from environment variables.
In the controllers folder, create a subscription controller with the following functions. First, a create subscription function that accepts a payment method ID and subscription tier from the user, creates a Stripe customer if one doesn't exist, attaches the payment method, creates a recurring monthly subscription in Stripe, and updates the user's subscription record in the database. Second, a webhook handler function that listens for Stripe webhook events including payment succeeded, payment failed, and subscription canceled. If payment succeeds, mark the subscription as active and unlock the account. If payment fails, record the failed payment timestamp and start the lockout countdown. If the account has not paid within seven days of the failed payment, set is locked to true and locked at to the current timestamp. Third, a cancel subscription function that cancels the subscription in Stripe and updates the database accordingly. Fourth, a get subscription status function that returns the current subscription tier, payment status, renewal date, and whether the account is locked.
In the middleware folder, create a lockout check middleware that runs on every protected route after auth middleware. If the user's subscription is locked, return a 403 response telling them their account is locked due to non-payment and they need to update their payment method.
In the routes folder, create a subscription routes file with POST routes for creating and canceling subscriptions, a GET route for subscription status, and a POST route for the Stripe webhook. The webhook route must not use the auth middleware since Stripe calls it directly.
Update server dot js to include the subscription routes. Output all files created.
