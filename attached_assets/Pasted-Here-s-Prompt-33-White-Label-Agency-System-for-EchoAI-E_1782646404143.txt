Here's Prompt 33: White Label Agency System for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system including Zapier integration. Now you need to add a White Label Agency System that allows marketing agencies and resellers to sell EchoAI under their own brand name, logo, and domain. This turns EchoAI into a platform that agencies can resell as their own product.
In the controllers folder create a white label controller with the following functions. First a create agency function that accepts agency name, logo URL, primary color, secondary color, custom domain, and support email, creates a new agency record, and links it to the authenticated admin user. Second a get agency settings function that returns the complete white label configuration for an agency. Third an update agency settings function that allows updating all white label settings. Fourth an add agency customer function that creates a new EchoAI customer account under the agency's umbrella with the agency's branding applied. Fifth a get agency customers function that returns all customers belonging to an agency with their subscription status and revenue. Sixth an agency revenue report function that calculates total monthly recurring revenue the agency is generating, number of active customers, and commission or markup earned.
In the models folder create a migration file that adds an agencies table with columns for agency ID, owner user ID, agency name, logo URL, primary color, secondary color, custom domain, support email, is active boolean, and timestamps. Also add an agency customers table linking agency ID to customer user ID with columns for agency ID, customer user ID, monthly price charged, and created at.
Create a white label middleware that checks incoming requests for a custom domain header or subdomain, looks up the matching agency configuration, and attaches the agency branding to the request so the frontend can apply it dynamically.
In the client source folder create a dynamic theming system that reads agency branding from the API and applies custom colors, logo, and company name throughout the dashboard. When a customer logs in through an agency's custom domain they should see the agency's branding everywhere instead of EchoAI branding. The login page, sidebar, and all headers should reflect the white label brand.
Add a White Label section to the admin panel that allows you as the platform owner to create and manage agency accounts, view all agencies and their customer counts, and see total revenue being generated through white label partnerships.
In the client source folder also add an Agency Portal section that is only visible to agency owner accounts. This section shows their customer list, monthly revenue, and a button to add new customers. It also shows their white label settings with a live preview of how their branded dashboard looks.
In the routes folder create agency routes with POST route for creating an agency, GET and PUT routes for agency settings, POST route for adding customers, GET route for agency customers, and GET route for revenue report. Protect all routes with auth and lockout middleware. Agency management routes should also require admin role.
Update server dot js to include agency routes at slash api slash agencies. Output all files created.
