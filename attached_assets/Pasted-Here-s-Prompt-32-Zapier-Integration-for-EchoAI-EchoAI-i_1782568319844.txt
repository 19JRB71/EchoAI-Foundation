Here's Prompt 32: Zapier Integration for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system including the Sales Script Generator. Now you need to add Zapier integration so EchoAI can connect with thousands of other tools that small business owners already use — like Google Sheets, Mailchimp, Calendly, Slack, QuickBooks, and hundreds more.
In the controllers folder create a Zapier controller with the following functions. First a create webhook function that accepts a trigger event name and a webhook URL from Zapier, stores it in a new webhooks table, and returns a webhook ID. Second a delete webhook function that removes a webhook subscription. Third a list webhooks function that returns all active webhooks for a brand. Fourth a trigger webhook function that is called internally by EchoAI whenever a trigger event occurs, loops through all webhooks subscribed to that event for that brand, and sends the event payload to each webhook URL via HTTP POST. Fifth a test webhook function that sends a sample payload to a webhook URL so Zapier can verify the connection works.
Create trigger events for the following EchoAI actions. New lead created. Lead temperature changed to hot. New campaign created. Campaign performance updated weekly. New review received. Review response posted. Sales script generated. Social media post published. Weekly report generated. Inbound call received. Outbound call completed.
In the models folder create a migration file that adds a webhooks table with columns for webhook ID, brand ID, event name, webhook URL, is active boolean, last triggered at, and timestamps. Also add a webhook delivery logs table with columns for log ID, webhook ID, event name, payload as JSON, response status, delivered at, and success boolean.
In the utils folder create a webhook dispatcher utility that handles sending payloads to webhook URLs with retry logic, logs each delivery attempt to the webhook delivery logs table, handles timeouts gracefully, and never blocks the main application thread.
Wire the webhook trigger function into the relevant existing controllers so events fire automatically. Wire new lead created into the lead controller. Wire lead temperature changed to hot into the chatbot controller. Wire weekly report generated into the scheduler. Wire inbound call received and outbound call completed into the phone controller. Wire new review received into the reputation controller.
In the client source folder add a Zapier Integration section to the existing dashboard sidebar. This section shows a list of all active webhooks with their trigger event and webhook URL. Show an Add Webhook button that opens a form asking for the trigger event from a dropdown of all available events and the webhook URL. Show a Test button next to each webhook and a Delete button. Include a link to EchoAI Zapier documentation placeholder.
In the routes folder create webhook routes with POST route for creating a webhook, DELETE route for removing a webhook, GET route for listing webhooks, and POST route for testing a webhook. All routes protected with auth and lockout middleware.
Update server dot js to include webhook routes at slash api slash webhooks. Output all files created.
