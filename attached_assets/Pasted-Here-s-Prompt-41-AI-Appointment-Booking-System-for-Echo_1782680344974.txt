Here's Prompt 41: AI Appointment Booking System for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system including feature gating and tier enforcement. Now you need to add an AI Appointment Booking System that allows the voice chatbot and phone agent to book appointments directly during conversations with hot leads, syncing automatically to the business owner's calendar with confirmation messages sent to both parties.
In the prompts folder create an appointment booking prompt file that defines the AI Appointment Scheduler Agent. This agent is invoked mid-conversation when a lead is scored as hot. It seamlessly transitions from the qualification conversation into scheduling without breaking the conversational flow. It checks the business owner's available time slots, presents two or three options naturally in conversation such as saying I have Tuesday at two or Thursday at ten available this week, which works better for you, confirms the appointment once the lead chooses, collects any additional information needed such as the lead's address for an in-person meeting or a specific topic for a consultation, and closes the conversation warmly with a clear summary of what was booked. The agent never sounds like a scheduling robot. It sounds like a helpful human assistant who happens to be very good at finding the right time.
In the controllers folder create an appointment controller with the following functions. First a get availability function that accepts a brand ID and a date range and returns available time slots based on the business owner's configured availability schedule and existing bookings. Second a book appointment function that accepts a lead ID, brand ID, proposed time, appointment type, and any notes, creates the appointment record, sends a confirmation email and SMS to the lead, sends a notification to the business owner, and if Google Calendar is connected creates the calendar event automatically. Third a get appointments function that returns all appointments for a brand with their status as scheduled, completed, cancelled, or no-show. Fourth an update appointment function that allows rescheduling or cancelling an appointment with automatic notification to the lead. Fifth a configure availability function that allows the business owner to set their weekly availability schedule such as Monday through Friday nine AM to five PM, block out specific dates, set appointment duration defaults such as thirty or sixty minutes, and set a buffer time between appointments such as fifteen minutes.
In the models folder create a migration file that adds an appointments table with columns for appointment ID, brand ID, lead ID, scheduled time, duration in minutes, appointment type, status, notes, google calendar event ID, confirmation sent boolean, and timestamps. Also add an availability schedules table with columns for schedule ID, brand ID, day of week, start time, end time, is available boolean, and timestamps. Also add an availability blocks table with columns for block ID, brand ID, blocked date, reason, and timestamps.
Wire the appointment booking agent into the chatbot controller so when a lead is scored as hot the chatbot naturally transitions into offering to schedule an appointment. Wire it into the phone controller so the AI phone agent can book appointments during calls. Make the transition seamless so it feels like one continuous conversation not a handoff to a different system.
Add Google Calendar integration in the config folder. When the business owner connects their Google account in settings the system uses the Google Calendar API to check real availability against their existing calendar events and creates new events when appointments are booked.
In the client source folder add an Appointments section to the existing dashboard sidebar. This section has three tabs. Tab one is the Calendar View showing a weekly calendar grid with all scheduled appointments displayed as colored blocks. Clicking an appointment shows full details including lead name, contact info, appointment type, notes, and status. Show a Mark Complete, Reschedule, and Cancel button on each appointment. Tab two is Availability Settings where the business owner sets their weekly schedule day by day with start and end times, can block out specific dates, sets default appointment duration, and sets buffer time between appointments. Include a toggle to connect Google Calendar. Tab three is Appointment History showing all past appointments with their outcome status and a link to the lead's full CRM profile.
Wire the availability configuration and Google Calendar connection into the existing Settings panel as well so business owners can manage it from either location.
Apply professional tier gate to appointment booking routes since this is a Professional and above feature.
Rebuild the React client after making changes. Output all files created and updated.
