Here's Prompt 52: AI Health Monitor and Screenshot Support System for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built a complete system with 51 prompts. Now you need to add an AI Health Monitor and Screenshot Support System that continuously watches every account for errors and issues, automatically fixes common problems silently, sends proactive notifications when something needs attention, and allows users to send a screenshot directly to an AI support agent that analyzes what they see and either fixes it or explains it in plain English.
In the prompts folder create a health monitor prompt file that defines two AI agents. First the AI Health Analyst Agent that accepts a health report for a brand including all recent errors, failed API calls, scheduler failures, disconnected integrations, expired tokens, failed campaigns, undelivered messages, and any other anomalies detected in the last twenty four hours. It analyzes the report, categorizes issues by severity as critical, warning, or info, identifies the root cause of each issue, determines which ones can be fixed automatically and which require user action, and generates a plain English summary the business owner can understand. Second the AI Screenshot Support Agent that accepts a base64 encoded screenshot image and a description of what the user is experiencing. It analyzes the screenshot visually, identifies what screen or feature is shown, identifies any visible errors, broken UI elements, or confusing states, and generates a clear plain English explanation of what is happening and exactly what the user should do to resolve it. If the issue can be fixed automatically it triggers the fix.
In the controllers folder create a health monitor controller with the following functions. First a run health check function that accepts a brand ID, queries every integrated system for recent errors and anomalies including checking Facebook API connection status, Twilio connection status, Stripe subscription status, email delivery failures, scheduler job failures, expired OAuth tokens, failed follow-up sequence touchpoints, failed SMS deliveries, failed webhook deliveries, and any database errors logged in the last twenty four hours, compiles a complete health report, calls the Anthropic API with the health analyst agent, and returns a categorized health report with recommended actions. Second an auto fix function that accepts a specific issue type and attempts to fix it automatically including refreshing expired tokens, retrying failed webhook deliveries, restarting stalled scheduler jobs, and clearing stuck sequences. Third a get health history function that returns the last thirty days of health check results for a brand. Fourth a screenshot support function that accepts a base64 encoded screenshot and user description, calls the Anthropic API with the screenshot support agent using vision capabilities, and returns a plain English analysis and recommended fix. Fifth a report issue function that saves a user reported issue with their screenshot to the database for tracking.
Create a continuous background health monitor that runs every hour using the existing scheduler infrastructure. For every active brand it runs a health check, attempts to auto-fix any fixable issues silently, and sends a notification to the business owner only if there is a critical issue that requires their attention or if an auto-fix attempt failed. Minor issues are fixed silently without bothering the user.
In the models folder create a migration file that adds a health checks table with columns for check ID, brand ID, check time, overall status as healthy, warning, or critical, issues found as JSON, issues auto-fixed as JSON, issues requiring attention as JSON, AI analysis text, and timestamps. Also add a support tickets table with columns for ticket ID, user ID, brand ID, screenshot URL, user description, AI analysis, resolution status, and timestamps.
In the client source folder add a Help and Support button in the bottom right corner of every dashboard page. This button sits above the existing Take the Tour button. When clicked it opens a support panel with two tabs. Tab one is Report an Issue showing a large Screenshot button that captures the current screen automatically using the browser's screen capture API, a text field where the user describes what they are experiencing, and a Send to AI Support button. When submitted the AI analyzes the screenshot and description and shows a response within seconds either explaining what to do or confirming it fixed the issue automatically. Tab two is Health Status showing the current health status for the selected brand as a simple green yellow or red indicator with a last checked timestamp, a list of any active issues with their severity and status, and a Run Health Check Now button.
Add a health status indicator to the top navigation bar showing a small colored dot next to the brand selector. Green means all systems healthy. Yellow means there are warnings. Red means there is a critical issue requiring attention. Clicking the dot opens the Health Status tab directly.
Wire the hourly health check into the existing scheduler. Wire the screenshot support into a new public-facing help widget that works even when the user is not logged in so they can get help from the login screen too.
Apply the health monitor to admin accounts so you can see the health status of every customer account from the admin panel. Add a Platform Health tab to the admin panel showing a summary of all accounts with their health status so you can instantly see if any customer is experiencing issues.
Rebuild the React client after making changes. Output all files created and updated.
