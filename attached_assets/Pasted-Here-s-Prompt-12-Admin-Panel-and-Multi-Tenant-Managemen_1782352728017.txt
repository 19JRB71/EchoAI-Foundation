Here's Prompt 12: Admin Panel and Multi-Tenant Management for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete customer-facing system. Now you need to build the admin panel that you, James, use to manage all customers, monitor the platform, handle billing issues, unlock accounts, and oversee everything from one central dashboard.
In the backend create an admin middleware file that checks if the authenticated user has an admin role and returns a 403 if they do not. Add a role column to the users table via a new migration with values of user and admin, defaulting to user.
In the controllers folder create an admin controller with the following functions. First a get all users function that returns a paginated list of all customers with their name, email, subscription tier, payment status, whether their account is locked, when they joined, and how many leads and campaigns they have. Second an unlock account function that manually unlocks a locked customer account and resets their payment failure countdown. Third a lock account function that manually locks a customer account. Fourth a get platform stats function that returns total number of customers, total active subscriptions, total revenue this month, total leads generated across all customers, total campaigns running, and total ad spend managed. Fifth an update user subscription function that allows you to manually change a customer's subscription tier. Sixth a delete user function that permanently removes a customer and all their data. Seventh a get user detail function that returns the complete profile of any customer including all their brands, campaigns, leads, and analytics.
In the client source folder create an admin panel that is only accessible to users with admin role. The admin panel has four sections. First an overview showing platform wide stats in metric cards. Second a customer list showing all customers in a searchable sortable table with columns for name, email, tier, payment status, account status, and actions. Actions include view details, unlock account, lock account, change tier, and delete. Third a customer detail view showing everything about one customer including their brands, campaigns, lead counts by temperature, and recent analytics. Fourth a platform health section showing the scheduler status, last time weekly analytics ran, last time optimization ran, and any system errors logged.
Add an admin seeder that creates your admin account automatically on first startup if it does not already exist using credentials from environment variables.
Update server dot js to include admin routes at slash api slash admin. All admin routes protected by both auth middleware and admin middleware. Output all files created.
