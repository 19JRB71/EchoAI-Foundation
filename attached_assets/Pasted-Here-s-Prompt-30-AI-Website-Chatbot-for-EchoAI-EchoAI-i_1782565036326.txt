Here's Prompt 30: AI Website Chatbot for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system including the AI Phone Agent. Now you need to add an AI Website Chatbot that business owners can embed on their own website. This chatbot lives on their website, engages visitors, answers questions about the business, qualifies leads, and captures contact information — all in the brand's voice and personality.
In the prompts folder create a website chatbot prompt file that defines the AI Website Chatbot Agent. This agent represents the business on their website. It greets visitors warmly, answers questions about the business using the brand profile, qualifies visitors using the same tire kicker, warm, and hot scoring system, captures their name, email, and phone number naturally during conversation, and books appointments if the business offers them. The agent's tone and personality matches the brand profile exactly. It never discusses competitors. It always steers the conversation toward a next step — a call, a visit, a purchase, or a form fill.
In the controllers folder create a website chatbot controller with the following functions. First a get chatbot config function that returns the public chatbot configuration for a brand including greeting message, brand name, accent color, and avatar style. Second a chat function that accepts a session ID, a message, and a brand ID, maintains conversation history by session, calls the Anthropic API with the website chatbot prompt and brand profile, scores the lead temperature, and returns the AI response. Third a capture lead function that saves the visitor's contact details to the leads table when they provide them during conversation. Fourth a get chatbot sessions function that returns all chatbot sessions for a brand with their transcript and lead outcome.
In the models folder create a migration file that adds a chatbot sessions table with columns for session ID, brand ID, lead ID, conversation history as JSON, temperature score, started at, and last active at.
Create a standalone embeddable chatbot widget as a single JavaScript file that business owners paste into their website. This widget creates a chat bubble in the bottom right corner of any webpage. When clicked it opens a clean chat interface. It connects to the EchoAI API using the brand ID. It works on any website regardless of what platform it was built on — WordPress, Squarespace, Wix, custom HTML, anything. The widget should be self contained with no external dependencies and load in under two seconds.
In the client source folder add a Chatbot Setup section to the existing dashboard. This section shows the embed code the business owner copies and pastes into their website, allows customizing the chatbot greeting message and accent color, shows a live preview of how the chatbot will look, and displays all recent chatbot sessions with their transcripts.
In the routes folder create chatbot routes with GET route for chatbot config which is public since any website can request it, POST route for chat which is public since website visitors are not logged in, POST route for capturing leads which is public, and GET route for chatbot sessions which is protected with auth and lockout middleware.
Update server dot js to include chatbot routes at slash api slash chatbot. Output all files created.
