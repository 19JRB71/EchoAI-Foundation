Here's Prompt 40: Feature Gating and Tier Enforcement for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system including team member invitations and role permissions. Now you need to enforce the three pricing tiers so customers can only access features included in their plan. This protects your revenue and creates a clear upgrade path.
The three tiers and pricing are as follows. Starter is one hundred dollars per month for one user with additional seats at fifty dollars per seat per month. Professional is three hundred fifty dollars per month for up to five users with additional seats at fifty dollars per seat per month beyond five. Enterprise is five hundred fifty dollars per month for unlimited users with no additional seat charges.
The platform admin account identified by the admin role in the users table is completely exempt from all feature gates, tier checks, billing calculations, team size limits, and subscription requirements. The admin sees the full platform with no restrictions at all times.
In the middleware folder create a feature gate middleware file that checks the authenticated user's subscription tier before allowing access to protected features. If the authenticated user has the admin role bypass all feature gates immediately and allow full access. For all other users the middleware accepts a required tier level and returns a 403 response with a clear upgrade message if the user's tier is below the required level. The 403 response must include the feature name, the user's current tier, the required tier to access it, the monthly base price of the required tier, and a message encouraging them to upgrade.
Create a tier configuration file in the config folder called tierConfig.js that defines the three tiers, their base prices, included seat counts, and maps every feature in EchoAI to its minimum required tier. Define tier levels as starter, professional, and enterprise in ascending order. Starter tier features include facebook ads, basic CRM, text chatbot, weekly reports, website chatbot embed, email notifications, and social media on two platforms. Professional tier features include everything in starter plus voice chatbot, AI phone agent, reputation management, sales scripts, content calendar, all six social platforms, Zapier integration, video scripts, and AI ad creative studio. Enterprise tier features include everything in professional plus white label agency, affiliate program, mobile API access, advanced analytics, customer feedback and surveys, and priority support.
Apply the feature gate middleware to the following routes. Add professional tier gate to voice routes at slash api slash voice, phone routes at slash api slash phone, reputation routes at slash api slash reputation, sales script routes at slash api slash sales-scripts, content calendar routes at slash api slash content-calendar, Zapier routes at slash api slash webhooks, video content routes at slash api slash video-content, and ad studio routes at slash api slash ad-studio. Add enterprise tier gate to agency routes at slash api slash agencies, affiliate routes at slash api slash affiliates, mobile v2 routes at slash api slash v2, and feedback routes at slash api slash feedback. Social routes at slash api slash social get a special gate that allows two platforms on starter and all six platforms on professional and above.
In the client source folder create a reusable FeatureGate component in the components folder. This component wraps any section, button, or feature. It accepts a required tier name as a prop. If the current user's tier meets the requirement it renders the children normally. If not it renders a clean upgrade prompt card instead. The upgrade prompt card shows a lock icon, the feature name, a one sentence description of what the feature does, the plan required with its monthly price, a list of three other features also included in that plan, and a large prominent Upgrade Now button that navigates to the subscription settings page. Style the card with a subtle gradient background and the lock icon in the tier's accent color. Make it feel aspirational not punitive.
Update the sidebar navigation component so every nav item that requires a higher tier than the current user's subscription displays a small lock badge with the tier name next to it. The item is still clickable but clicking it shows the FeatureGate upgrade prompt instead of the feature. Admin accounts see no lock badges ever.
Update the landing page pricing section to accurately reflect the three tiers. Starter at one hundred dollars per month, Professional at three hundred fifty dollars per month, Enterprise at five hundred fifty dollars per month. Show additional seats at fifty dollars per seat per month on Starter and Professional. Show the accurate feature list for each tier. Add a comparison table below the pricing cards showing which features are in which tier so prospects can clearly see the value of upgrading. Make the Enterprise tier visually stand out as the most popular or best value option.
Update the onboarding subscription selection step to show the correct prices, accurate feature lists per tier, and the per seat pricing note so new customers understand exactly what they are paying.
Wire team size enforcement so the system checks seat limits when a team member is invited. Starter accounts attempting to add a second user are shown the per seat pricing and asked to confirm the fifty dollar monthly charge before proceeding. Professional accounts adding a sixth or more user are shown the same confirmation. Enterprise accounts have no limit and no confirmation required. Admin accounts have no restrictions.
Rebuild the React client after all changes. Output all files created and updated.
