Here's Prompt 10: AI Campaign Optimization and Competitor Intelligence for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've already built the full system including authentication, billing, Facebook campaigns, brand discovery, lead qualification, analytics, and the customer dashboard. Now you need to add the AI Campaign Optimization engine and Competitor Intelligence system that automatically analyzes what competitors are doing and uses that intel to improve your customers' campaigns.
In the prompts folder create a campaign optimization prompt file that defines two AI agents. First a competitor analysis agent that accepts a business niche, competitor names or URLs, and target audience description, searches for publicly available information about what competitors are running in terms of messaging, positioning, and marketing angles, identifies gaps and opportunities competitors are not exploiting, and returns a structured competitor intelligence report. Second a campaign optimization agent that accepts the brand profile, current campaign performance data, competitor intelligence report, and historical analytics, analyzes what is working and what is not, generates three new ad creative variations with copy and image direction tailored to the brand voice, recommends audience targeting adjustments, recommends budget allocation changes, and explains its recommendations in plain language the business owner can understand.
In the controllers folder create an optimization controller with the following functions. First a run competitor analysis function that accepts brand ID and a list of competitor names or website URLs, calls the Anthropic API with the competitor analysis agent prompt, stores the results in a new competitor intelligence table, and returns the structured report. Second an auto optimize campaigns function that pulls the brand profile, latest analytics, active campaigns, and most recent competitor intelligence report, calls the Anthropic API with the campaign optimization agent prompt, applies the recommended changes to active Facebook campaigns via the Facebook API, generates three new ad creative variations and queues them for testing, and logs all changes made to the campaigns table. Third a get optimization history function that returns a log of all past optimizations for a brand including what changes were made and the performance impact.
In the models folder create a migration file that adds a competitor intelligence table with columns for intelligence ID, brand ID, competitor names as a JSON array, intelligence report as JSON, and timestamps. Also add an optimization history table with columns for optimization ID, brand ID, campaign ID, changes made as JSON, performance before as JSON, performance after as JSON, and timestamps.
In the routes folder create an optimization routes file with POST route for running competitor analysis, POST route for triggering auto optimization, and GET route for optimization history. All routes protected with auth and lockout middleware.
Update server dot js to include optimization routes at slash api slash optimize. Also update the Monday morning scheduler to automatically run auto optimize campaigns for every active brand after recording weekly analytics so the system continuously improves without any manual intervention.
Output all files created.
