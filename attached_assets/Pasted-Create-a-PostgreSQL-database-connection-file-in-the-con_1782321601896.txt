Create a PostgreSQL database connection file in the config folder that connects to a local PostgreSQL database. Then create a schema file that defines eight tables with their columns and relationships. Table one is Users, which stores account info including user ID, email, password hash, subscription tier, team size, created date, and updated date. Table two is Subscriptions, which tracks billing cycles, payment method, subscription tier, renewal dates, payment status, and lockout logic—if payment fails after X days, the account locks completely. Table three is Brands, which stores the business profile data including brand ID, user ID, brand personality, voice description, visual style preferences, and target audience info. Table four is Leads, which logs every prospect including lead ID, brand ID, lead name, email, phone, temperature category as tire kicker, warm, or hot, conversation history, and conversion status. Table five is Campaigns, which tracks Facebook ad campaigns including campaign ID, brand ID, campaign name, budget, performance metrics like cost per lead and conversion rate, ad creative variations tested, and launch date. Table six is CRMInteractions, which records every touchpoint including interaction ID, lead ID, interaction type as chatbot conversation or website visit, timestamp, and interaction details. Table seven is APIIntegrations, which stores secure credentials including integration ID, user ID, platform as Facebook or Stripe, API token encrypted, and connection status. Table eight is Analytics, which aggregates weekly reporting data including analytics ID, brand ID, week date, total spend, total leads, cost per lead, conversions, and return on ad spend.

Define relationships between these tables using foreign keys. Users have many Brands, Campaigns, and Subscriptions. Brands have many Leads and Campaigns. Leads have many CRMInteractions. Output the complete SQL schema as a migration file that can be run to create all tables at once. Include proper data types, constraints, indexes on frequently queried columns, and timestamps for created and updated dates on every table.
