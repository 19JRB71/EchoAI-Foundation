Here's Prompt 23: Mobile PWA Setup for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system. Now you need to make EchoAI work as a Progressive Web App so customers can install it on their phone like a native app, get push notifications for hot leads, and use it smoothly on mobile devices.
In the client public folder create a web app manifest file that defines the EchoAI app name, short name, description, theme color as dark, background color, display mode as standalone so it feels like a native app, start URL, and icon definitions in multiple sizes including 192x192 and 512x512. Create simple placeholder icon files in those sizes using canvas or SVG that show the EchoAI logo letters on a dark background with a cyan accent color.
In the client source folder create a service worker file that caches the app shell and static assets on install so the app loads instantly even on slow connections, serves cached assets when offline, and handles push notification events by showing a notification with the lead name and temperature when a hot lead comes in.
In the client source folder create a push notification utility that requests notification permission from the user on first login, registers the service worker, subscribes to push notifications using the Web Push API, and sends the subscription endpoint to the backend so the server knows where to send push notifications for that user.
In the backend create a push notification controller with the following functions. First a save subscription function that accepts a push subscription object from the browser and saves it to a new push subscriptions table linked to the user. Second a send push notification function that accepts a user ID and a notification payload and sends a push notification to all of that user's registered devices using the web-push library. Third wire the hot lead alert system so that whenever a lead is scored as hot it sends both an email notification and a push notification to the business owner instantly.
In the models folder create a migration file that adds a push subscriptions table with columns for subscription ID, user ID, endpoint, keys as JSON, and timestamps.
Update the client index HTML to link the web manifest and register the service worker. Rebuild the React client after making changes. Output all files created.
