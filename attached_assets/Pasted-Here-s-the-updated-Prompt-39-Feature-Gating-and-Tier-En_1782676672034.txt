Here's the updated Prompt 39: Feature Gating and Tier Enforcement for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system with 38 prompts worth of features. Now you need to enforce the three pricing tiers so customers can only access features included in their plan. This protects your revenue and creates a clear upgrade path.
The three tiers and pricing are as follows. Starter is one hundred dollars per month for one user with additional seats at fifty dollars per seat per month. Professional is three hundred fifty dollars per month for up to five users with additional seats at fifty dollars per seat per month beyond five. Enterprise is five hundred fifty dollars per month for unlimited users with additional seats at fifty dollars per seat per month. Every tier's base price includes the first seat and the per seat add-on applies when team size exceeds the included seats.
Important: The platform owner admin account identified by the ADMIN_EMAIL environment variable must have permanent full access to every feature across all tiers with no charges, no restrictions, and no feature gates applied. The admin account is exempt from all tier checks, billing calculations, team size limits, and subscription requirements. This allows the platform owner to use EchoAI personally for their own business at no cost while maintaining full platform management capabilities.
In the middleware folder create a feature gate middleware file that checks the authenticated user's subscription tier before allowing access to protected features. If the authenticated user is the platform admin identified by the admin role in the users table, bypass all feature gates immediately and allow full access. For all other users the middleware accepts a required tier level and returns a 403 response with a clear upgrade message if the user's tier is below the required level. The response should include the feature name, the user's current tier, the required tier to access it, and the monthly price of the required tier so the user knows exactly what they need to upgrade to.
Create a tier configuration file in the config folder that maps every feature in EchoAI to its required tier. Starter tier includes Facebook ad automation, basic CRM, lead qualification text chatbot, weekly reports, website chatbot embed, email notifications, and social media posting on two platforms only. Professional tier includes everything in Starter plus voice chatbot, AI phone agent with Twilio, reputation management, sales script generator, content calendar, all six social platforms, Zapier integration, video script generator, and AI ad creative studio. Enterprise tier includes everything in Professional plus white label agency system, affiliate program, mobile app API access, advanced analytics, API marketplace access, customer feedback and survey system, and priority support.
Apply the feature gate middleware to the following route groups. Apply professional tier gate to voice routes, phone routes, reputation routes, sales script routes, content calendar routes, social routes beyond two platforms, Zapier webhook routes, video content routes, and ad studio routes. Apply enterprise tier gate to agency routes, affiliate routes, mobile v2 routes, and feedback routes. Admin accounts bypass all of these gates automatically.
Update the Stripe subscription and billing logic so the monthly charge is calculated correctly based on tier base price plus fifty dollars multiplied by the number of additional seats beyond what is included in the base. Starter base includes one seat so additional seats are any users beyond one. Professional base includes five seats so additional seats are any users beyond five. Enterprise base includes unlimited seats so there are no additional seat charges on Enterprise. When a user adds a team member the system automatically calculates the new monthly total and updates the Stripe subscription accordingly. The admin account is never included in any billing calculation.
Update the subscription controller so when a user upgrades their tier the system immediately unlocks the new features without requiring a logout. When a user downgrades their tier the system locks the higher tier features at the next billing cycle. These rules do not apply to the admin account.
In the client source folder create a feature gate component that wraps any dashboard section or button. When a user tries to access a gated feature they see a clean upgrade prompt instead of the feature. The upgrade prompt shows what the feature does, which plan includes it, the monthly price including per seat costs, and a prominent Upgrade Now button that takes them directly to the subscription management page. Make the upgrade prompt visually appealing and persuasive so upgrading feels exciting not restrictive. Admin accounts never see upgrade prompts and always see the full dashboard with no locks or restrictions.
Update the sidebar navigation so features above the user's current tier are visible but marked with a lock icon and the plan name required. Clicking a locked sidebar item shows the upgrade prompt instead of the feature. Admin accounts see no lock icons and have full unrestricted sidebar navigation.
Update the landing page pricing section to show the three tiers with their correct base prices of one hundred dollars, three hundred fifty dollars, and five hundred fifty dollars per month. Show the accurate feature list for each tier. Include a note that additional seats are fifty dollars per seat per month on all plans. Make the pricing section clear, honest, and compelling.
Update the onboarding subscription selection step to show the correct prices, feature lists, and per seat pricing so new customers understand exactly what they are paying before they enter their card.
Wire the team size enforcement so Starter accounts are blocked from adding more than one user without being prompted to upgrade or pay for an additional seat, Professional accounts are blocked from adding more than five users without the additional seat charge applying, and Enterprise accounts have no user limit. Admin accounts have no team size restrictions.
