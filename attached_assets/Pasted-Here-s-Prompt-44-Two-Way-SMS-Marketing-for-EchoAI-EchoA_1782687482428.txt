Here's Prompt 44: Two-Way SMS Marketing for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system including the sidebar redesign. Now you need to add Two-Way SMS Marketing that allows business owners to send bulk AI-written SMS campaigns to their leads and customers, receive replies, and have the AI automatically respond to incoming messages in the brand's voice.
In the prompts folder create an SMS marketing prompt file that defines two AI agents. First the SMS Campaign Writer Agent that accepts a brand profile, campaign goal, target audience segment, and a call to action, and generates five SMS message variations each under one hundred sixty characters, punchy and compelling, written in the brand's voice, with a clear call to action and a compliant opt-out instruction. Second the SMS Auto-Reply Agent that accepts a brand profile, an incoming SMS message from a lead or customer, and the lead's conversation history, and generates a natural conversational reply that continues the conversation, answers questions, and steers toward the desired outcome such as booking an appointment or visiting a link.
In the controllers folder create an SMS marketing controller with the following functions. First a create campaign function that accepts a brand ID, campaign name, message content, and a list of lead IDs or a segment filter such as all hot leads or all warm leads, creates the campaign record, and queues the messages for sending. Second a send campaign function that executes a queued campaign by sending the SMS to every recipient using the existing Twilio integration, tracking delivery status for each message. Third a get campaigns function that returns all SMS campaigns for a brand with their status, recipient count, delivery rate, and reply rate. Fourth a get campaign detail function that returns a specific campaign with per-recipient delivery and reply status. Fifth an inbound SMS handler function that Twilio calls when a lead replies to an SMS, identifies which brand and lead the message belongs to, calls the AI auto-reply agent, sends the reply automatically, updates the lead's conversation history, and fires the hot lead alert if the conversation scores as hot. Sixth a get conversations function that returns all two-way SMS conversations for a brand showing the full message thread with each lead. Seventh an opt out handler that processes STOP messages and marks leads as opted out so they never receive SMS again.
In the models folder create a migration file that adds an SMS campaigns table with columns for campaign ID, brand ID, campaign name, message content, segment filter, status as draft, sending, sent, or failed, recipient count, delivered count, reply count, scheduled at, sent at, and timestamps. Also add an SMS messages table with columns for message ID, campaign ID if applicable, brand ID, lead ID, direction as inbound or outbound, message body, Twilio message SID, delivery status, sent at, and timestamps. Also add an SMS opt outs table with columns for opt out ID, brand ID, phone number, opted out at, and timestamps.
Wire the inbound SMS handler as a public Twilio webhook at slash api slash sms slash inbound that requires no authentication since Twilio calls it directly. Wire opt out checking into every outbound SMS send across the entire platform so opted out numbers never receive any SMS from any feature including follow-up sequences, appointment reminders, and survey invitations.
In the client source folder add a new SMS Marketing section to the existing dashboard sidebar under the Marketing group. This section has four tabs. Tab one is Campaigns showing all SMS campaigns in a table with their name, status, recipient count, delivered count, reply rate, and sent date. Show a Create Campaign button that opens a campaign builder. The campaign builder lets users select their brand, name the campaign, choose a recipient segment from dropdowns including all leads, hot leads only, warm leads only, or specific leads selected from a list, enter a campaign topic or goal, click Generate Messages to see five AI-written variations, select their preferred variation or edit it, preview the final message with character count, and send immediately or schedule for later. Tab two is Conversations showing all two-way SMS threads in a list on the left and the selected conversation thread on the right similar to a standard messaging interface. Show the lead name, their temperature badge, and last message preview in the list. Show the full chronological message thread on the right with timestamps. Allow the business owner to type and send manual replies from this interface. Tab three is Contacts showing all leads with their SMS opt-in status. Show opted out leads with a red badge and allow manually re-subscribing if the lead requests it. Tab four is Analytics showing total messages sent this month, average delivery rate, average reply rate, and a simple chart of SMS activity over the last thirty days.
Apply professional tier gate to SMS marketing routes since this is a Professional and above feature.
Rebuild the React client after making changes. Output all files created and updated.
