Here's Prompt 6: Brand Management and Discovery Agent for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've already built the project structure, database schema, authentication, Stripe billing, and Facebook campaign agent. Now you need to build the Brand Management system and the Brand Discovery Agent — the three-part conversational AI that gets to know each business owner deeply so EchoAI can represent their brand authentically.
In the controllers folder, create a brand controller with the following functions. First, a create brand function that accepts a brand name and creates a new brand record in the brands table linked to the authenticated user. Second, a get brands function that returns all brands belonging to the authenticated user. Third, an update brand function that allows updating brand name, personality, voice description, visual style preferences, and target audience. Fourth, a delete brand function that removes a brand and all associated data. Fifth, a get brand profile function that returns the complete brand profile including all discovery data.
In the prompts folder, create a brand discovery prompt file that defines the three-part conversational AI agent. Part one opens with a warm professional greeting, explains that EchoAI wants to get to know their brand deeply so it can represent them authentically, then asks one open-ended question about what their business does and the problem it solves. It listens to the response and engages genuinely with what they said before moving to part two. Part two digs deeper based on their previous answers, asking about brand personality, tone of voice, visual preferences, and target audience. It responds conversationally to what they share and asks natural follow-up questions. Part three synthesizes everything learned across all three exchanges, confirms the brand profile back to the user in plain language, and asks if it accurately represents them. If they confirm, save the complete profile. If they want changes, update accordingly. The agent's tone throughout is professional and friendly, knowledgeable but warm. It never asks more than two questions at a time and always responds to what the user actually said before asking anything new.
In the controllers folder, add a brand discovery controller that manages the three-part conversation flow, stores conversation state between exchanges, calls the Anthropic API to power the conversational agent using the brand discovery prompt, and saves the completed brand profile to the brands table when the user confirms.
In the routes folder, create a brand routes file with POST route for creating a brand, GET route for all brands, GET route for a single brand profile, PUT route for updating a brand, DELETE route for deleting a brand, and POST route for the brand discovery conversation. All routes protected with auth and lockout middleware.
Update server dot js to include brand routes at slash api slash brands. Output all files created.
