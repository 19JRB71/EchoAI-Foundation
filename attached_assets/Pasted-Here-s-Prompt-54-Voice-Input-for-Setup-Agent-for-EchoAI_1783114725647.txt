Here's Prompt 54: Voice Input for Setup Agent for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built a complete system including the AI Sales Agent. Now you need to add voice input to the Setup Agent so new customers can speak their answers out loud instead of typing. This is the feature that makes EchoAI completely hands free — you just talk to Echo and it builds your entire marketing account automatically. No keyboard required except to approve connections.
In the client source folder update the SetupAgent component to add voice input capability. Add a large microphone button below the answer text field on every question screen. When the user taps or clicks the microphone button it starts recording their voice using the Web Speech API or the existing MediaRecorder infrastructure already built in the voice chatbot. While recording show a pulsing red indicator and the text Listening. When they stop speaking or click the microphone button again it stops recording, transcribes their speech, and populates the answer text field with the transcribed text. The user can then review what Echo heard, edit it if needed, and click Next to submit. If the transcription is wrong they can just speak again and it overwrites the previous transcription. Show a small text below the microphone button that says Tap to speak or Click to speak depending on whether they are on mobile or desktop.
Support two transcription methods with automatic fallback. First use the Web Speech API which is built into Chrome, Edge, and Safari and requires no backend calls — it transcribes in real time as the user speaks and is the fastest option. Second if the Web Speech API is not available fall back to the existing OpenAI Whisper integration already built in the voice chatbot system, sending the recorded audio to the backend for transcription. Detect which method is available on page load and use the best one automatically. Show a small indicator telling the user which mode is active — Instant voice recognition for Web Speech API or Voice transcription for Whisper fallback.
Add a voice mode toggle at the top of the Setup Agent that lets users switch between voice mode and text mode. Default to voice mode on mobile devices where typing is harder and text mode on desktop where typing is easier. Store the preference in localStorage so it persists.
Update the Setup Agent greeting screen to mention voice input. After the initial welcome message add a line that says You can speak your answers or type them — whichever feels more natural.
In the backend add a transcription endpoint specifically for the Setup Agent voice input at slash api slash setup-agent slash transcribe that accepts an audio blob, calls the existing OpenAI Whisper integration, and returns the transcribed text. This is the fallback endpoint used when the Web Speech API is not available. Reuse the existing audio handling infrastructure from the voice chatbot system.
Make the voice input work smoothly on mobile. Use large touch targets for the microphone button. Show clear visual feedback during recording. Auto-submit after a natural pause in speaking of two seconds so mobile users do not have to tap twice. On desktop require an explicit click to stop recording so users have time to think.
Update the Setup Agent consent screen to mention that if the user chose voice mode their spoken answers were processed to set up their account and that they can switch to text mode at any time from the top of the screen.
Rebuild the React client after making changes. Output all files created and updated.
