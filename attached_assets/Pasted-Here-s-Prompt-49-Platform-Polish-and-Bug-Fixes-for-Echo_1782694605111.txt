Here's Prompt 49: Platform Polish and Bug Fixes for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built a complete 48-prompt system. Now you need to do a comprehensive platform polish and bug fix pass to make sure everything is production ready, performant, secure, and professional before the final guided tour prompt.
Start by doing a complete audit of the entire codebase. Check every backend route, controller, middleware, and utility for the following issues and fix them all. First check for any missing error handling where a crash could take down the server and add try-catch blocks where they are missing. Second check for any database queries that are missing proper indexes for frequently queried columns and add them. Third check for any API endpoints that are missing input validation and add proper validation with clear error messages. Fourth check for any memory leaks or blocking operations in the scheduler that could degrade performance over time and fix them. Fifth check for any hardcoded values that should be environment variables and move them. Sixth check for any console.log statements left in production code that should be removed or replaced with proper logging. Seventh verify that every route that should be protected with auth and lockout middleware actually has it and add it where missing.
On the frontend do a complete audit of the React client for the following issues. First check every API call for proper loading states and error handling and add them where missing so users never see a blank screen or unhandled error. Second check every form for proper validation before submission and add clear error messages where missing. Third check the mobile responsiveness of every section and fix any sections that break on small screens. Fourth check for any components that make unnecessary re-renders and optimize them. Fifth remove any placeholder text, fake data, or TODO comments left in the UI and replace them with proper empty states. Sixth verify that all tier-gated features show proper upgrade prompts instead of broken UI for lower tier users. Seventh make sure every section has a proper empty state with helpful guidance when there is no data yet instead of just showing nothing.
Do a security audit of the entire platform. First verify that all user inputs are sanitized before being inserted into the database. Second verify that all file uploads are validated for type and size. Third verify that rate limiting is in place on authentication endpoints to prevent brute force attacks. Fourth verify that all JWT tokens are properly validated and expired tokens are rejected. Fifth verify that the Stripe webhook signature validation is in place and working. Sixth verify that all Twilio webhook signature validation is in place. Seventh check for any SQL injection vulnerabilities and fix them. Eighth verify that CORS is properly configured for production.
Do a performance audit. First check the Monday morning scheduler to make sure all jobs run efficiently and none can block others if they fail. Second check that all database queries use parameterized queries consistently. Third verify that the React client bundle is optimized and not loading unnecessary code. Fourth check that images from DALL-E are handled efficiently and not causing memory issues.
Fix the pre-existing bug noted during build where PUT slash api slash brands slash id returns 500 with invalid input syntax for type json when targetAudience is sent as a plain string. Add proper JSON parsing and validation for all JSONB fields across all controllers.
Update the dot env example file to make sure every environment variable used anywhere in the codebase is documented with a clear description of what it is and where to get it.
Generate a complete production deployment checklist as a markdown file called DEPLOYMENT.md at the root of the project. This checklist should cover every step needed to deploy EchoAI to production including setting up environment variables, running database migrations in order, configuring Stripe webhooks, configuring Twilio webhooks, setting up a custom domain, configuring email SMTP, setting up Firebase for push notifications, and any other steps needed to go live.
