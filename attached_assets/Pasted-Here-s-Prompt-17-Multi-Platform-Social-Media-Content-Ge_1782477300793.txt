Here's **Prompt 17: Multi-Platform Social Media Content Generation and Posting for EchoAI**:

EchoAI is an AI-powered SaaS marketing platform. You've built the complete system including Facebook ad automation, brand discovery, lead qualification, analytics, and the customer dashboard. Now you need to build the Social Media Content Generation and Posting system that automatically creates and schedules posts across all major social media platforms.

In the controllers folder create a social media controller with the following functions. First, a connect social account function that accepts a platform name and authentication credentials, securely stores the credentials in the api integrations table, and verifies the connection works. Supported platforms are Facebook, Instagram, TikTok, LinkedIn, Twitter, and YouTube. Second, a generate social content function that accepts a brand ID and content topic or theme, uses the Anthropic API with the brand profile to generate five unique social media post variations, each tailored specifically for that platform with appropriate tone, length, hashtags, and formatting. Third, a schedule post function that accepts a post, platform, and scheduled time, and queues the post to be published at that exact time. Fourth, a publish post function that actually publishes the queued post to the specified platform using that platform's API. Fifth, a get social calendar function that returns all scheduled and published posts for a brand across all platforms in a calendar view. Sixth, a get post performance function that pulls engagement metrics from each platform showing likes, comments, shares, and reach for each posted content.

In the prompts folder create a social content generation prompt file that defines the AI agent. The agent accepts a brand profile, a content topic, and a target platform. It generates five unique post variations optimized for that specific platform considering character limits, hashtag conventions, visual descriptions, call to action buttons, posting times that perform best, and the brand's unique voice and personality. For Facebook and Instagram the agent suggests image or video descriptions. For TikTok it suggests trending sounds and hashtags. For LinkedIn it optimizes for professional tone. For Twitter it keeps it short and punchy. For YouTube it generates video title and description ideas. Each variation should feel native to that platform while staying true to the brand.

In the models folder create a migration file that adds a social posts table with columns for post ID, brand ID, platform, post content, scheduled time, published time, status, and engagement metrics. Also add a social accounts table with columns for account ID, brand ID, platform, platform username, and encrypted credentials.

In the routes folder create a social routes file with POST route for connecting a social account, POST route for generating social content, POST route for scheduling a post, GET route for the social calendar, and GET route for post performance metrics. All routes protected with auth and lockout middleware.

Create a scheduler function that runs every minute and publishes any posts that have reached their scheduled time. Wire this into the existing scheduler system.

Update server dot js to include social routes at slash api slash social. Output all files created.
