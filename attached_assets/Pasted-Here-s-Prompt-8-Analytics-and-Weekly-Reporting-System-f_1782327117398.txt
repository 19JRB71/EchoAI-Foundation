Here's Prompt 8: Analytics and Weekly Reporting System for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've already built the project structure, database schema, authentication, Stripe billing, Facebook campaign agent, brand discovery agent, and lead qualification chatbot. Now you need to build the Analytics and Weekly Reporting system that aggregates performance data and delivers simple weekly summaries to each customer.
In the controllers folder, create an analytics controller with the following functions. First, a record weekly analytics function that pulls all active campaigns for a brand from the campaigns table, fetches the latest performance data from Facebook for each campaign, calculates total spend, total leads, cost per lead, conversions, and return on ad spend for the week, and saves a new record to the analytics table. Second, a get analytics function that returns all weekly analytics records for a brand sorted by most recent first. Third, a get current week summary function that returns the most recent analytics record for a brand in a simple human readable format showing total spend, total leads, cost per lead, and conversions. Fourth, a generate weekly report function that uses the Anthropic API to generate a friendly professional weekly summary email for the business owner that explains their results in plain language, highlights what's working, notes any areas of concern, and suggests one or two simple next steps. The tone should match the brand's voice profile.
In the utils folder, create a scheduler utility that uses node-cron to automatically run the record weekly analytics function for every active brand every Monday morning at eight AM. This ensures every customer gets their data updated automatically without any manual intervention.
In the controllers folder, add a reporting controller that manages the weekly report generation, calls the Anthropic API with the brand profile and latest analytics data, formats the report as a clean simple email, and logs that the report was sent in the CRM interactions table.
In the routes folder, create an analytics routes file with GET route for all analytics for a brand, GET route for the current week summary, POST route to manually trigger a weekly report generation, and POST route to manually trigger analytics recording. All routes protected with auth and lockout middleware.
Update package dot json to add node-cron as a dependency. Update server dot js to include analytics routes at slash api slash analytics and initialize the cron scheduler on startup. Output all files created.
