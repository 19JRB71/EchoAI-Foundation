Here's Prompt 35: Native iOS/Android App Preparation for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system including the Affiliate Program. Now you need to prepare EchoAI for native mobile apps on iOS and Android. This means restructuring the API to be fully mobile-ready, creating a React Native compatible API layer, and building the mobile app foundation that can be submitted to the Apple App Store and Google Play Store.
In the backend create a dedicated mobile API version at slash api slash v2 slash that is optimized for mobile clients. This version should return leaner response payloads without unnecessary fields, support pagination on all list endpoints using cursor-based pagination for better mobile performance, include a mobile-specific authentication flow that supports biometric login tokens and longer JWT expiry of thirty days for mobile sessions, and return standardized response envelopes with status, data, message, and pagination fields on every endpoint.
In the controllers folder create a mobile auth controller with the following functions. First a mobile login function that accepts email and password, returns a thirty day JWT token, and includes a refresh token for seamless re-authentication without requiring the user to log in again. Second a refresh token function that accepts a refresh token and returns a new access token. Third a biometric token function that generates a short-lived biometric token that the mobile app stores in the device's secure enclave and uses for Face ID or fingerprint login.
In the models folder create a migration file that adds a refresh tokens table with columns for token ID, user ID, token hash, device ID, device name, expires at, and created at.
Create a mobile API documentation file that describes every available endpoint, request format, response format, and authentication method. This will be used by a React Native developer to build the iOS and Android apps.
In the utils folder create a mobile response formatter utility that wraps all mobile API responses in the standard envelope format and handles cursor-based pagination consistently across all endpoints.
Create a React Native project scaffold in a new folder called EchoAI-Mobile at the root level. This scaffold should include the basic project structure, navigation setup using React Navigation, authentication flow screens for login, biometric login, and registration, a home dashboard screen showing the three key metrics, a leads screen showing the CRM with temperature badges, a notifications screen for hot lead alerts, and a settings screen. Use React Navigation for screen navigation and AsyncStorage for local data persistence. Include placeholder screens for all major features that can be filled in by a React Native developer.
Add push notification support to the backend. Create a push notifications controller that accepts a device push token from the mobile app, stores it in a new device tokens table, and sends push notifications using Firebase Cloud Messaging when hot leads come in, when weekly reports are ready, and when payments fail.
In the routes folder create mobile routes at slash api slash v2 slash auth for mobile authentication and slash api slash v2 slash push for push notification registration. Output all files created.
