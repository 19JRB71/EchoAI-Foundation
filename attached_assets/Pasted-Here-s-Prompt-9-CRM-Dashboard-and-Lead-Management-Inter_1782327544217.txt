Here's Prompt 9: CRM Dashboard and Lead Management Interface for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've already built the project structure, database schema, authentication, Stripe billing, Facebook campaign agent, brand discovery agent, lead qualification chatbot, and analytics reporting. Now you need to build the frontend dashboard that business owners log into to see their leads, campaigns, analytics, and CRM data in one simple clean interface.
Build a single page web application using React and Tailwind CSS. The dashboard should have five main sections.
Section one is the login and registration screen. Clean simple form where users enter email and password to log in or register. On successful login store the JWT token and redirect to the main dashboard.
Section two is the main dashboard overview. Show three simple metric cards at the top displaying total spend this week, total leads this week, and cost per lead this week. Below that show a simple bar chart of leads generated over the last four weeks. Keep it clean and easy to read. No clutter.
Section three is the leads CRM panel. Show a list of all leads with their name, email, phone, temperature category displayed as a colored badge — red for tire kicker, yellow for warm, green for hot — conversion status, and date created. Allow filtering by temperature category. Allow clicking a lead to see their full profile and conversation history.
Section four is the campaigns panel. Show all active Facebook campaigns with their name, budget, cost per lead, conversion rate, and status. Allow manually triggering a campaign optimization from this panel.
Section five is the settings panel. Allow users to update their profile, manage their subscription, connect their Facebook account, and view their brand profile. Include a button to start or restart the brand discovery conversation.
The overall design should be clean, modern, and simple. Use a dark sidebar for navigation between the five sections. Main content area is light. Mobile responsive. Professional but not overly corporate. This is a tool for small business owners so keep everything straightforward and easy to understand.
Wire the frontend to all existing backend API endpoints. Use fetch for API calls with the JWT token in the authorization header on all protected routes. Handle loading states and errors gracefully. Output all files created.
