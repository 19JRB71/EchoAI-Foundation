Here's Prompt 48: Customer Intelligence Engine for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system including the Advanced ROI Dashboard. Now you need to build the Customer Intelligence Engine — the brain of EchoAI that continuously analyzes every piece of data from every channel for each business, builds a growing intelligence profile specific to that business, and uses accumulated knowledge to make every future AI recommendation smarter and more personalized over time.
In the prompts folder create a customer intelligence prompt file that defines the AI Customer Intelligence Agent. This agent accepts a complete intelligence profile for a business including all historical performance data, lead patterns, conversion patterns, audience insights, channel effectiveness, content performance, competitor positioning, customer feedback themes, and seasonal trends. It synthesizes everything into a growing strategic intelligence document that gets smarter every week. It identifies the business's ideal customer profile based on who actually converts, the best performing content angles based on what actually gets engagement, the optimal posting times and channels based on real performance data, the most effective follow-up timing based on response patterns, pricing and offer insights based on conversion data, and competitor gaps the business can exploit. It generates five strategic recommendations ranked by potential impact that are specific, actionable, and grounded in the business's own data. Each recommendation includes what to do, why the data supports it, and exactly how to implement it using EchoAI's existing features.
In the controllers folder create a customer intelligence controller with the following functions. First a build intelligence profile function that accepts a brand ID, pulls data from every channel across the entire platform including all campaigns, leads, conversions, calls, SMS conversations, email performance, social media engagement, appointment outcomes, feedback scores, competitor intelligence, ROI snapshots, and follow-up sequence performance, synthesizes it into a comprehensive intelligence profile JSON, and stores it in the database. Second an analyze and recommend function that takes the current intelligence profile, calls the Anthropic API with the customer intelligence agent, and returns five prioritized strategic recommendations. Third a get intelligence profile function that returns the current intelligence profile and latest recommendations for a brand. Fourth an update intelligence function that is called by the scheduler to rebuild the intelligence profile every week with fresh data, compare it to the previous week's profile to identify trends and trajectory, and generate a new set of recommendations. Fifth a get intelligence history function that returns the last twelve weeks of intelligence profiles so the business owner can see how their strategic position has evolved over time. Sixth an apply recommendation function that accepts a recommendation ID and automatically implements it within EchoAI where possible — for example if the recommendation is to post more on Instagram it creates a content calendar entry, if it is to follow up faster it adjusts the follow-up sequence timing, if it is to target a different audience it updates the campaign targeting.
In the models folder create a migration file that adds a customer intelligence table with columns for intelligence ID, brand ID, week date, raw profile data as JSON, recommendations as JSON, trends identified as JSON, trajectory score from one to ten, AI analysis text, and timestamps. Also add an applied recommendations table with columns for application ID, intelligence ID, brand ID, recommendation text, action taken, applied at, outcome notes, and timestamps.
Wire the intelligence profile builder into the existing Monday morning scheduler so every brand gets a fresh intelligence analysis every week automatically. The intelligence engine runs last in the scheduler after all other weekly jobs have completed so it has access to the freshest data from that week.
In the client source folder add a new Customer Intelligence section to the existing dashboard sidebar under the Business group. This section has four tabs.
Tab one is the Intelligence Brief showing this week's strategic intelligence document. At the top show a trajectory score displayed as a gauge or large number with a trend arrow showing whether the score improved or declined from last week. Below show the five strategic recommendations as large cards ranked by impact. Each card shows the recommendation title, a one paragraph explanation grounded in the brand's specific data, an impact level as high, medium, or low, an implementation complexity as easy, medium, or hard, and two buttons — Apply Automatically where possible and Mark as Done for manual actions. Show a Regenerate Analysis button at the top.
Tab two is the Intelligence Profile showing the raw synthesized intelligence data in a readable format. Show sections for Ideal Customer Profile, Best Performing Content Angles, Optimal Channel Mix, Follow-Up Timing Insights, Competitive Positioning, and Seasonal Trends. Each section shows the AI's findings based on the brand's actual data.
Tab three is Trends showing a twelve week history of trajectory scores as a line chart, a table of how each metric has trended over time, and a comparison of this week's recommendations versus last week's to show how the strategy is evolving.
Tab four is Applied Recommendations showing a log of all recommendations that have been applied, what action was taken, and any outcome notes the business owner has added.
Apply enterprise tier gate to customer intelligence routes since this is the most advanced Enterprise feature and represents EchoAI's strongest competitive advantage.
Wire the Customer Intelligence Engine into the existing onboarding flow so new Enterprise customers see a message explaining that their intelligence engine is warming up and will deliver its first strategic brief after one week of data collection.
Rebuild the React client after making changes. Output all files created and updated.
