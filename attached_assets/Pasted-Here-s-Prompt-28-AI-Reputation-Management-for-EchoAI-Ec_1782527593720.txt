Here's Prompt 28: AI Reputation Management for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system including the ROI dashboard. Now you need to add AI Reputation Management so EchoAI automatically monitors and responds to customer reviews on Google, Facebook, and Yelp in the business owner's brand voice.
In the prompts folder create a reputation management prompt file that defines the AI Review Response Agent. This agent accepts a brand profile, a review text, the reviewer's name, the star rating, and the platform. It generates a professional, personalized response that matches the brand's voice and personality. For five star reviews it expresses genuine gratitude and reinforces what the customer loved. For three and four star reviews it thanks them, acknowledges their feedback, and explains how the business is improving. For one and two star reviews it responds calmly and professionally, acknowledges the concern, apologizes sincerely, and offers to make it right with contact information. The response should never sound defensive or automated. It should feel like it was written personally by the business owner.
In the controllers folder create a reputation controller with the following functions. First a fetch reviews function that pulls recent reviews from Google Business Profile using the existing Google integration, Facebook using the existing Facebook integration, and a manual entry option for Yelp since Yelp does not have a public API. Second a generate response function that accepts a review and calls the Anthropic API with the review response agent prompt to generate a personalized response. Third a post response function that posts the generated response back to Google or Facebook using their APIs. Fourth a save review function that saves reviews and their responses to a new reviews table. Fifth a get reviews function that returns all reviews for a brand with their response status showing pending, responded, or ignored.
In the models folder create a migration file that adds a reviews table with columns for review ID, brand ID, platform, reviewer name, star rating, review text, response text, response status, posted at, and timestamps.
In the client source folder add a new Reputation Management section to the existing dashboard sidebar. This section has two tabs. Tab one is the Review Inbox showing all reviews across all platforms in a unified feed. Each review shows the platform icon, reviewer name, star rating displayed as colored stars, review text, date, and response status badge. Reviews with no response show a Generate Response button. Clicking it shows the AI generated response in an editable text box so the owner can tweak it before posting. Show a Post Response button and an Ignore button. Tab two is Reputation Stats showing average star rating per platform, total reviews, percentage of reviews responded to, and a simple rating trend over the last thirty days.
Wire all endpoints to the backend. Rebuild the React client after making changes. Output all files created.
