Here's Prompt 53: AI Sales Agent with Three-Way Call Support for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built a complete system including the AI Health Monitor. Now you need to add an AI Sales Agent that handles inbound demo calls automatically, qualifies prospects, answers questions about EchoAI, closes sales, and supports a three-way call mode where the human owner joins as a co-pilot and can say Hey Echo to get the AI to answer questions in real time.
In the prompts folder create a sales agent prompt file that defines two AI agents. First the AI Sales Agent that knows everything about EchoAI including every feature, every pricing tier, every use case, and every competitive advantage over GoHighLevel and other marketing platforms. It handles inbound demo calls professionally and warmly, introduces itself as Echo the AI assistant for EchoAI, qualifies the prospect by asking about their business type, current marketing challenges, and goals, explains how EchoAI solves their specific problems using the information they just shared, handles common objections such as price, complexity, and skepticism about AI, and attempts to close the sale by offering to start a free setup right now or schedule a follow-up call with James. Its tone is confident, warm, knowledgeable, and never pushy. Second the Three-Way Co-Pilot Agent that activates when it detects the trigger phrase Hey Echo during a live call. It listens to the conversation context, understands the question being asked, and provides a concise confident answer the human can relay to the prospect or that plays directly into the call. It specializes in technical questions, pricing edge cases, and objection handling that the human might not have memorized.
In the controllers folder create a sales agent controller with the following functions. First an initiate demo call function that accepts an inbound call from a prospect via Twilio, connects them to the AI Sales Agent, and begins the qualification and sales conversation. Second a handle sales conversation function that manages the back and forth conversation between the AI and the prospect, maintaining full conversation context, scoring the prospect's interest level from one to ten throughout the call, and detecting buying signals. Third a three way call invite function that sends an SMS to the platform owner with a join link when a prospect reaches an interest score of seven or above, allowing the owner to join the call as a co-pilot. Fourth a detect hey echo function that monitors the call audio for the trigger phrase Hey Echo and when detected pauses the main conversation, processes the context, generates a co-pilot response, and either reads it aloud on the call or sends it as a text to the owner's phone depending on configuration. Fifth a call summary function that when the call ends generates a complete summary including prospect name and contact info, business type, pain points identified, interest score, objections raised, outcome as booked demo, follow-up scheduled, or not interested, and recommended next steps. Sixth a get sales calls function that returns all sales calls with their summaries and outcomes. Seventh a book demo function that when a prospect agrees to move forward sends them a calendar link, creates a follow-up task, and triggers the existing follow-up sequence system.
In the models folder create a migration file that adds a sales calls table with columns for call ID, prospect phone, prospect name, business type, conversation history as JSON, interest score, outcome, call duration, summary, booked demo boolean, follow-up scheduled boolean, and timestamps.
Create a sales agent configuration panel where the platform owner configures the AI Sales Agent. Configuration options include the owner's phone number for three-way call invites, whether Hey Echo responses are read aloud on the call or sent as SMS to the owner, the calendar booking link for demo scheduling, a list of the top five objections and preferred responses that the AI should use, and a toggle to enable or disable the sales agent.
In the client source folder add a Sales Agent section to the admin panel since this feature is specifically for the platform owner not for customers. This section has four tabs. Tab one is Live Calls showing any currently active sales calls with the prospect name, how long they have been talking, current interest score displayed as a colored meter, and a Join Call button that triggers the three-way call invite. Tab two is Call History showing all past sales calls in a table with prospect name, business type, interest score, outcome, call duration, and a View Summary button. Clicking View Summary shows the full AI-generated call summary. Tab three is Configuration showing all the sales agent settings including owner phone, Hey Echo response mode, booking link, and custom objection responses. Tab four is Performance showing total calls this month, average interest score, conversion rate from call to booked demo, and a breakdown of common objections raised.
Wire the inbound call handling to a Twilio phone number dedicated to EchoAI sales. When someone calls that number they reach the AI Sales Agent. This is separate from the existing phone agent system which handles customer business calls.
Output all files created and updated.
