Here's Prompt 42: AI Follow-Up Sequences for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system including the AI Appointment Booking System. Now you need to add AI Follow-Up Sequences that automatically contact leads via email, SMS, and phone on a set schedule until they respond, book an appointment, or convert. This turns every lead into a fully automated follow-up machine that never lets a prospect fall through the cracks.
In the prompts folder create a follow-up sequence prompt file that defines the AI Follow-Up Sequence Designer Agent. This agent accepts a brand profile, the lead's temperature category, the lead's conversation history from the chatbot or phone agent, and the desired outcome such as book an appointment, close a sale, or get a callback. It designs a complete multi-step follow-up sequence with up to seven touchpoints spread across fourteen days. Each touchpoint specifies the channel as email, SMS, or phone call, the delay in hours or days after the previous touchpoint, and the message content written in the brand's voice. The sequence escalates naturally — starting warm and helpful, becoming more direct as days pass, and ending with a final breakup message that leaves the door open. Messages reference what the lead said in their original conversation so they feel personal not automated. The sequence stops automatically the moment the lead responds, books an appointment, or converts.
In the controllers folder create a follow-up sequence controller with the following functions. First a generate sequence function that accepts a lead ID and desired outcome, pulls the lead's full conversation history and brand profile, calls the Anthropic API with the follow-up sequence designer prompt, and returns a complete personalized sequence. Second a save and activate sequence function that saves the generated sequence and immediately activates it so the first touchpoint fires at the right time. Third a pause sequence function that pauses an active sequence without deleting it. Fourth a cancel sequence function that permanently stops a sequence. Fifth a get sequences function that returns all sequences for a brand with their status and progress. Sixth a get sequence detail function that returns a specific sequence with all its touchpoints and their delivery status. Seventh an execute touchpoint function that is called by the scheduler to execute each touchpoint — sending the email via the existing email system, sending the SMS via the existing Twilio integration, or triggering an outbound call via the existing phone agent.
Create a smart stop logic system that monitors all active sequences for a brand and automatically stops a sequence when any of the following events occur — the lead replies to an email, the lead responds to an SMS, the lead answers a phone call, the lead books an appointment, or the lead's status is updated to converted in the CRM. This ensures leads are never contacted after they have already engaged.
In the models folder create a migration file that adds a follow-up sequences table with columns for sequence ID, brand ID, lead ID, desired outcome, status as draft, active, paused, completed, or cancelled, current step, total steps, started at, completed at, and timestamps. Also add a sequence touchpoints table with columns for touchpoint ID, sequence ID, step number, channel as email, SMS, or phone, delay hours, scheduled at, executed at, delivery status, message content, and timestamps.
Wire the auto-stop logic into the lead controller so updating a lead to converted status cancels all active sequences for that lead. Wire it into the chatbot controller so a lead responding to the chatbot cancels their active sequence. Wire it into the phone controller so a lead answering a call cancels their active sequence. Wire it into the appointment controller so booking an appointment cancels the sequence.
Update the existing Monday morning scheduler to also check every five minutes for sequence touchpoints that are due and execute them. Use a separate cron job running every five minutes for sequence execution so touchpoints fire close to their scheduled time.
Wire automatic sequence creation into the lead qualification flow. When a lead is scored as warm automatically generate and activate a warm lead nurture sequence with the goal of booking an appointment. When a lead is scored as hot and does not immediately book an appointment generate and activate a hot lead closing sequence with the goal of closing the sale.
In the client source folder add a Follow-Up Sequences section to the existing dashboard sidebar. This section has three tabs. Tab one is Active Sequences showing all currently running sequences with the lead name, sequence goal, current step out of total steps, next touchpoint channel and scheduled time, and a Pause and Cancel button. Tab two is Sequence Builder where users can manually select a lead, choose a desired outcome, click Generate Sequence to see the AI-designed touchpoint plan displayed as a visual timeline, and then activate it. Tab three is Sequence History showing all completed and cancelled sequences with their outcome.
Apply professional tier gate to follow-up sequence routes since this is a Professional and above feature.
Rebuild the React client after making changes. Output all files created and updated.
