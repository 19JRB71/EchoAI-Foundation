Here's Prompt 64: Permanent Login, Music Integration, Faster Echo, and Voice Execution for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built Echo with always-on voice conversation. Now you need to add permanent login so Echo is always ready, music playback so Echo can play songs on command, faster response times so Echo feels instant, and voice execution so when you say go Echo acts immediately without any extra confirmation steps for routine actions.
PART 1 — PERMANENT LOGIN
Extend the JWT token expiry to 30 days instead of the current short expiry. Store the token in localStorage so it persists across browser sessions. When the user opens EchoAI on their regular device Echo is already logged in and ready without requiring a password entry. The user only needs to log in again if they have been inactive for 30 days, if they explicitly log out, or if they open EchoAI on a new device for the first time.
Add a remember this device toggle on the login screen. When checked the token persists for 30 days. When unchecked the token expires when the browser closes. Default to checked so most users stay logged in automatically.
When Echo detects the user returning after being away for more than 8 hours treat it as a new session and play the morning wake up music and deliver the morning briefing automatically even without a fresh login. Echo should feel like it has been waiting for you and is glad you are back.
PART 2 — MUSIC INTEGRATION
Build a music integration that allows Echo to play music on voice command. Integrate with YouTube Music or Spotify using their embed APIs to play music directly inside EchoAI without leaving the platform.
When the user says "Hey Echo play some AC/DC" Echo searches for AC/DC music and starts playing it. When the user says "Hey Echo play my morning playlist" Echo plays an upbeat energetic playlist to start the day. When the user says "Hey Echo turn it up" Echo increases the volume. When the user says "Hey Echo skip this song" Echo skips to the next track. When the user says "Hey Echo pause the music" Echo pauses. When the user says "Hey Echo stop the music" Echo stops completely.
Build a small music player widget that appears in the bottom of the sidebar when music is playing. Show the current song name, artist, album art, and playback controls. Keep it compact so it does not distract from the main dashboard.
When Echo is about to speak during a briefing or alert automatically fade the music volume down so Echo's voice is clear, then fade the music back up after Echo finishes speaking.
Add a morning music ritual feature. The user can tell Echo what music they want to wake up to each morning. When the user logs in or returns after 8 hours Echo plays their chosen morning music for 30 to 60 seconds before transitioning into the morning briefing. The user can skip the music at any time by saying "Hey Echo skip the music" or "Hey Echo let's get started."
PART 3 — FASTER ECHO RESPONSES
Implement streaming audio so Echo starts speaking before the full response is generated. Instead of waiting for ElevenLabs to generate the complete audio file Echo starts playing the first few words while the rest is still being generated. This makes Echo feel instant and responsive.
Cache common Echo responses so frequently spoken phrases like good morning, your appointment starts in 15 minutes, and I have completed that task play instantly from cache without any API call.
Optimize the wake word response time so Echo activates within half a second of hearing "Hey Echo." The activation tone should play immediately while Echo processes what comes next.
Pre-generate the morning briefing audio in the background starting at 6am so when the user logs in the briefing is ready to play instantly with no generation delay.
Pre-load the morning wake up music so it plays immediately on login with no buffering delay.
PART 4 — VOICE EXECUTION
When the user gives Echo a verbal go ahead Echo executes immediately without any extra confirmation steps for routine actions.
Define routine actions that execute immediately on voice approval. These include launching a campaign that was already previewed and approved, posting content that was already reviewed, sending a follow-up message that is within the approved sequence, adjusting a campaign budget within the approved guardrails, booking an appointment that was already discussed, and marking a lead as converted.
Define high-stakes actions that always require an explicit confirmation regardless of verbal approval. These include spending more than the monthly budget limit, deleting any data, sending mass communications to more than 100 people at once, and making any change that cannot be easily reversed.
When the user says "yes" or "go ahead" or "let's do it" or "do it" or "absolutely" or "sounds good" Echo immediately executes the pending routine action and confirms verbally what it just did.
When a high-stakes action is pending Echo says clearly "This one needs your explicit confirmation because it cannot be easily undone. Say confirm to proceed or cancel to stop." Only the word confirm executes a high-stakes action.
PART 5 — ECHO PROACTIVE CHANNEL SUGGESTIONS
Build a system where Echo proactively notices marketing channels and tools that could benefit the owner's businesses but are not currently being used and suggests them naturally in conversation.
Echo monitors what channels competitors are using that the owner is not. If Echo notices competitors getting traction on Pinterest and the owner has no Pinterest presence Echo mentions it naturally during the weekly briefing. "James I noticed three of your competitors are running Pinterest campaigns targeting the same audience as Blacor Homes. Home building inspiration is one of Pinterest's top categories. Want me to look into adding it to your marketing mix?"
Echo suggests tools and integrations when it notices gaps. If the owner never uses the Zapier integration Echo might say "James I noticed you have not connected any of your other business tools through Zapier yet. If you use tools like QuickBooks or Calendly I can connect them and automate a lot of the manual work between them. Want me to show you what is possible?"
Echo tracks its own suggestions and never repeats the same suggestion within 30 days. If the owner declines a suggestion Echo notes it and does not bring it up again for 90 days.
Wire all voice execution into the existing Echo companion message pipeline. Wire the music player into the existing sidebar layout. Wire the streaming audio into the existing ElevenLabs voice system. Wire the proactive suggestions into the existing weekly intelligence briefing. Output all files created and updated.
