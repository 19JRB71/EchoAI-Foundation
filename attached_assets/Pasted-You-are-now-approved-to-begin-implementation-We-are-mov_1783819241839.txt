PHASE 4 — PREVENT DUPLICATE PAID REQUESTS

Implement exact duplicate protection first.

Generate a request fingerprint using:

* Business ID

* Task type
* Feature
* Provider or provider category
* Model configuration
* System prompt version
* Business-knowledge version
* Normalized request
* Important parameters
* Time window when the answer is time-sensitive

If the same request is currently in progress:

* Do not start another provider call.
* Join the existing request.
* Return the same completed result to all waiting callers.

If the same request was recently completed and is still valid:

* Return the cached result.

Protect against:

* Double-clicks
* Multiple form submissions
* Browser refreshes
* React duplicate events
* Repeated webhooks
* Scheduler overlap
* Multiple server instances
* Retry loops
* Repeated voice commands
* Deployment restarts

Use a database-backed or distributed lock. Do not rely only on local server memory.
