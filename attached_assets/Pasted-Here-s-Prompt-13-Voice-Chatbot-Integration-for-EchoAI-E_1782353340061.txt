Here's Prompt 13: Voice Chatbot Integration for EchoAI:
EchoAI is an AI-powered SaaS marketing platform. You've built the complete system including the lead qualification chatbot. Now you need to add voice capability so the lead qualification chatbot can have real spoken conversations with prospects instead of text only. This is the feature that makes EchoAI stand out from every competitor.
In the backend create a voice controller with the following functions. First a generate speech function that accepts text and a voice style, calls a text to speech API to convert the AI chatbot response into natural sounding audio, and returns the audio file. Second a transcribe speech function that accepts an audio file uploaded by the prospect, calls a speech to text API to transcribe what they said, and returns the transcribed text. Third a voice chat function that combines both — accepts audio from the prospect, transcribes it, passes the transcription to the lead qualification chatbot controller, gets the AI response, converts that response to speech, and returns the audio back to the prospect. This creates a seamless voice conversation loop.
Use the OpenAI Whisper API for speech to text transcription since it is the most accurate available. Use the OpenAI text to speech API for converting AI responses to natural sounding audio. Store the OpenAI API key in environment variables.
In the client source folder create a voice chat component that shows a simple clean interface with a microphone button in the center. When the prospect taps or clicks the microphone button it records their voice. When they release it sends the audio to the backend voice chat endpoint. While waiting for the response show a subtle animated indicator. When the response comes back play the audio automatically. The interface should feel natural and conversational like talking to a real person. Show a transcript of the conversation below the microphone button so prospects can read along if they want.
Add a toggle in the lead qualification chatbot so it can operate in either text mode or voice mode. Default to voice mode on mobile devices and text mode on desktop but allow switching.
In the routes folder create voice routes with POST route for speech to text, POST route for text to speech, and POST route for the full voice chat loop. The voice chat route should be public since prospects are not logged in. All other voice routes protected with auth middleware.
Update server dot js to include voice routes at slash api slash voice. Add multer as a dependency for handling audio file uploads. Output all files created.
